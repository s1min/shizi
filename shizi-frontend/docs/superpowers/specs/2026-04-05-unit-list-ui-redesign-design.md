# Unit List 页面 UI 改版设计

日期：2026-04-05
页面：`src/pages/unit-list/index.vue`
目标：在不改变页面核心信息结构与交互语义的前提下，对学习单元页进行一次现代化 UI 改版，并临时保留两套视觉方案用于对比，最终确认后删除未选中的一套。

## 1. 背景与目标

当前页面已经具备基础分组、摘要、状态与操作能力，但 UI 上仍有几个明显问题：

- 顶部 `wd-navbar` 为纯白背景，固定区氛围较弱，首屏识别度不够。
- 字库摘要模块存在冗余信息与重复分组，“当前字库”文案与四宫格统计块显得略碎。
- `StatusLegendBar` 单独占据一行，但状态信息已经在卡片中反复表达，信息重复。
- `UnitTaskCard` 的状态角标仍偏传统标签样式，和卡片本身没有形成统一视觉。
- 星级奖励与操作区分离，底部操作按钮样式偏旧，整体产品感不够强。
- 页面背景偏暖但略显厚重，需要进一步变浅、变轻。

本次改版目标：

1. 提升页面现代感与精致度。
2. 保留儿童教育产品应有的亲和感，不做“冷硬后台风”。
3. 统一顶部区域、摘要卡、单元卡、按钮体系的视觉语言。
4. 用两套主题方案对比 UI 气质：
   - 方案 A：奶油轻拟物
   - 方案 B：精致现代
5. 对比完成后，仅保留一套样式，另一套直接删除，不做长期双主题维护。

## 2. 设计决策

### 2.1 总体策略

采用“**统一结构 + 两套视觉主题对比**”的策略。

也就是说：

- 页面布局结构只改一套。
- 组件结构只改一套。
- 临时引入两套视觉样式（例如页面根节点类名或主题变量）。
- 最终选中一套后，删除另一套样式分支。

**为什么采用这个策略：**
- 用户本次需求核心是对比 UI 气质，而不是比较两套完全不同的信息架构。
- 统一结构能避免“布局差异”干扰最终判断。
- 后续删除成本低，只需收敛主题变量和样式分支，不需要回收两套页面实现。

### 2.2 两套主题定义

#### 方案 A：奶油轻拟物
关键词：柔和、亲和、轻盈、暖感、儿童教育产品氛围更强。

主要特征：
- navbar 使用暖白、浅杏、极淡品牌色的轻渐变。
- 卡片保留轻微浮起感与柔和阴影。
- 按钮圆润、饱满，带轻按压感。
- 状态角标像融入卡片的软标签。
- 背景以接近白色的奶油底为主，减少厚重黄感。

#### 方案 B：精致现代
关键词：克制、利落、现代、专业、层级清晰。

主要特征：
- navbar 使用白到极淡品牌色的低对比渐变。
- 卡片更平整，弱化阴影，强调留白和对齐。
- 按钮更简洁，主次关系更干净。
- 状态角标像结构化信息徽标，而非独立组件。
- 大面积背景更接近白色，仅在重点信息区使用淡色。

## 3. 布局与信息结构

### 3.1 页面整体结构

页面调整为“**固定顶部区 + 轻量内容流**”结构：

- 顶部固定区：承载 navbar、tabs 与顶部氛围背景。
- 内容滚动区：承载字库摘要、错字模式提示、单元分组列表。
- 内容区增加足够的顶部内边距，避免被固定顶部遮挡。

### 3.2 顶部区域改造

#### 现状问题
- `wd-navbar` 背景纯白，存在感弱。
- tabs 像独立悬浮组件，顶部区域整体感不足。

#### 新设计
- 为 navbar 外层增加固定吸顶容器。
- 顶部容器增加渐变背景与底部轻分隔。
- tabs 容器和 navbar 形成一个整体视觉模块。
- 滚动时 navbar 不跟随页面内容滚动。

#### 两个主题的差异
- A：更柔和、更有暖感，顶部带轻微“奶油高光”氛围。
- B：更克制、更清爽，强调干净分层和现代控制条感。

## 4. 字库摘要模块（LibrarySummaryCard）

### 4.1 结构调整

现有 `LibrarySummaryCard` 做如下改造：

1. 删除眉标题“当前字库”。
2. 当前字库名称直接作为模块主标题。
3. 描述文案保留，但弱化占比。
4. “切换字库”按钮升级为现代胶囊式工具按钮，并增加 `wd-icon` 的 `swap` 图标。
5. `summary-row` 保留统计信息，但整体样式精简，不再强调“四块小卡片拼接感”。

### 4.2 信息层级

模块内部层级调整为：
- 第一层：字库名（最强）
- 第二层：切换字库按钮（工具动作）
- 第三层：一句说明文案（弱化）
- 第四层：统计信息块（轻量辅助）

### 4.3 summary-row 优化原则

- 保留现有统计项，不改变业务含义。
- 视觉上更精简，减少边框感和分割感。
- 将“数据块”设计为轻量信息单元，而不是独立卡片。
- 仍然遵循 2x2 布局，避免引入新的认知成本。

#### 主题差异
- A：统计块用淡彩色底与轻内发光，保持温润。
- B：统计块趋向纯净、扁平、弱边界，更偏产品化 dashboard-lite。

## 5. 图例移除（StatusLegendBar）

### 决策
`StatusLegendBar` 从页面中移除，不再展示独立图例条。

### 原因
- 当前状态已经在摘要统计、卡片角标、进度条、按钮文案中多次表达。
- 图例条单独占空间，但对完成任务帮助有限。
- 移除后可显著提升首屏简洁度。

### 替代方式
- 状态颜色统一收敛到卡片内部：
  - 卡片角标
  - 进度条/状态条
  - 必要时的局部状态色
- 通过结构内表达代替额外说明模块。

## 6. 单元卡片（UnitTaskCard）

### 6.1 顶部状态角标改造

#### 现状问题
- 当前使用 `wd-tag`，与卡片视觉割裂。
- 存在边框与标签组件感，现代感不足。

#### 新设计
- 移除独立标签组件表达。
- 改为卡片右上角融合式状态角标。
- 状态角标直接贴合卡片顶部区域，作为卡片头部的一部分。
- 不使用边框，依靠背景色、文字色和圆角切面表达状态。

#### 状态表达原则
- 未学习：中性灰
- 学习中：品牌蓝系
- 待测试：暖橙系
- 已测试 / 已清空：绿色系
- 待强化：偏暖红橙系

### 6.2 内容层级优化

卡片内容顺序保持为：
1. 单元标题
2. lesson 文案 / 错字数量辅助信息
3. 汉字预览主内容
4. 进度状态条
5. 底部奖励与操作区

改造重点：
- 单元标题与状态角标形成稳定头部结构。
- 汉字预览维持为卡片主视觉焦点。
- 进度条颜色更轻，不抢主内容。

### 6.3 stars 与 actions 同行

现有 stars 在 tested 状态下单独占一行。改为：
- stars 与 `unit-card-actions` 同一行。
- stars 放在左侧，按钮组放在右侧。
- stars 尺寸放大，增强奖励反馈感。
- 若不是 tested 状态，则该区域只保留按钮，布局自然回流。

### 6.4 操作按钮升级

按钮体系统一为：
- 主按钮：主要推进动作（如重新测试 / 去强化）
- 次按钮：辅助动作（如复习）

按钮设计原则：
- 同高、同圆角、同排布逻辑。
- 主按钮更具强调感，但不使用过重阴影。
- 次按钮弱化处理，但保持足够点击感。
- 按钮整体更现代，减少旧式描边按钮的生硬感。

#### 主题差异
- A：按钮更圆润饱满，有轻微胶囊感与柔和阴影。
- B：按钮更利落，依靠色块和留白建立层级。

## 7. 分组标题（UnitStageSection）

`UnitStageSection` 保持原有“分组标题 + 列表”的结构，但做轻量优化：

- stage accent 保留，但更精炼。
- 标题行间距与列表间距重新统一。
- 标题视觉不应强于卡片内容，只承担分组作用。

#### 主题差异
- A：accent 可保留轻渐变与更柔和圆角。
- B：accent 更细更克制，突出信息效率。

## 8. 色彩与背景策略

### 8.1 全页背景

统一原则：**背景尽量淡，颜色尽量只服务于重点信息。**

- 页面背景从当前偏暖底色进一步提亮。
- 保留淡淡主题氛围，但避免整体泛黄。
- 使用局部淡色块烘托信息，而不是大面积重色。

### 8.2 颜色角色

颜色仅按角色使用：
- 品牌主色：主要按钮、学习中状态、关键交互
- 中性色：正文、辅助文案、默认模块
- 状态色：待测试 / 已测试 / 待强化等

### 8.3 两套主题的颜色方向

方案 A：
- 主背景：暖白、奶油白
- 强调辅助色：浅杏、薄荷绿、淡橙、极淡蓝
- 整体观感：柔和、亲和

方案 B：
- 主背景：接近白色
- 强调辅助色：品牌蓝 + 低饱和状态色
- 整体观感：清爽、干净、利落

## 9. 技术实现建议

### 9.1 结构改动范围

涉及文件：
- `src/pages/unit-list/index.vue`
- `src/pages/unit-list/components/LibrarySummaryCard.vue`
- `src/pages/unit-list/components/UnitTaskCard.vue`
- `src/pages/unit-list/components/UnitStageSection.vue`

移除页面引用：
- `src/pages/unit-list/components/StatusLegendBar.vue`（页面内不再使用）

### 9.2 临时对比实现方式

建议在页面根节点加主题类（示例命名，不要求完全一致）：
- `theme-cream`
- `theme-modern`

实现原则：
- 通过根节点类切换主题变量与局部样式分支。
- 避免复制两份组件模板。
- 结构一致，确保对比只来自视觉语言。

### 9.3 样式组织原则

- 优先把颜色、阴影、背景、按钮风格收敛到 CSS 变量层。
- 结构类与主题类解耦。
- 与布局相关的尺寸继续遵循项目规则：优先使用 8 的倍数，必要时使用 4 的倍数。

## 10. 验收标准

改版完成后，页面应满足：

1. navbar 为固定顶部，不随内容滚动。
2. navbar 背景不再是单调纯白，具备符合主题的浅渐变氛围。
3. `LibrarySummaryCard` 中去掉“当前字库”。
4. “切换字库”按钮升级为更现代的图标按钮风格。
5. `summary-row` 比当前版本更精简，保留信息但减少碎片感。
6. `StatusLegendBar` 被移除。
7. `UnitTaskCard` 状态角标位于右上角，与卡片融为一体，不使用边框标签样式。
8. `stars` 模块更大，且与 `unit-card-actions` 在同一行。
9. `unit-card-actions` 按钮更现代，主次关系明确且风格统一。
10. 全页背景更淡，整体颜色更贴合主题色并降低厚重感。
11. 页面存在两套可切换的视觉方案，仅用于 UI 对比；最终会删除未选方案。

## 11. 非目标

以下内容不在本次设计范围内：
- 不新增业务字段。
- 不调整页面核心交互流程。
- 不改变单元分组逻辑与数据来源。
- 不重构 `useUnitListPage` 的业务逻辑。
- 不引入长期的双主题系统。

## 12. 推荐结论

从产品气质上看：
- 如果更强调儿童教育的亲和感与温度，优先考虑方案 A（奶油轻拟物）。
- 如果更强调现代感、通用性与后续全站视觉统一，优先考虑方案 B（精致现代）。

本次实现将同时提供两套视觉方案进行直接对比，帮助快速决策，最终只保留一套。
