# 学习相关页面儿童化整改设计

## 背景
当前项目中的学习相关页面已经完成了基础导航补齐，但在实际运行中暴露出更深层的问题：页面不仅存在微信胶囊遮挡风险，而且顶部操作模型、视觉层级和儿童友好性整体还不够成熟。

用户进一步反馈后，希望从专业儿童线上教育产品经理和专业 UI 设计师的角度，对学习页面及学习相关页面进行整体整改，而不是继续做局部导航补丁。这个方向是正确的，因为儿童教育产品中的学习流页面，不应只解决“能不能退出”，更要解决“孩子看什么、家长控制什么、页面如何减少误触和认知负担”。

## 产品判断
### 1. 学龄前儿童不是页面控制的主要承担者
学龄前儿童关注的是：字形、图片、声音、按钮反馈；
家长关注的是：学习进度、是否可退出、是否误触、是否还能回到正确流程。

因此，学习相关页面必须明确区分：
- 儿童主视觉区：只承载学习任务
- 家长辅助控制区：提供弱化但稳定的退出能力

### 2. 沉浸式学习页不应把“退出”放在主视觉轴上
无论是右上角 `✕`，还是顶部中间“退出学习”，本质上都不适合学龄前儿童学习场景：
- 对儿童来说不易理解
- 对家长来说位置不稳定或效率不高
- 容易打断沉浸感

更合理的设计是：
- 顶部优先传递“当前做到哪里了”
- 退出入口放在稳定、弱化、但易于家长识别的位置

### 3. 学习相关页面需要形成统一语义
学习页、测试页、复习页、单元完成页应属于同一学习流，但目前头部模式、视觉语义、按钮优先级还不统一。

本次整改目标不是把它们做成完全一样，而是让它们具备同一套规则：
- 顶部优先展示进度
- 退出入口不抢主任务
- 主按钮始终是当前任务的下一步
- 成就页突出奖励和下一步，而非返回

## 目标
- 提升学习相关页面对学龄前儿童的可理解性和可操作性
- 降低误触和任务打断风险
- 为家长提供稳定、可预期的辅助退出入口
- 统一学习页、测试页、复习页、完成页的视觉和交互语义
- 给出可直接落地的页面级整改方案与原型说明

## 整体设计原则
### 1. 孩子看内容，家长管退出
页面中最醒目的信息只能是：
- 当前学习内容
- 当前操作按钮
- 当前奖励/反馈

退出入口只能作为辅助控制项存在。

### 2. 顶部只强调进度，不强调操作
顶部区域主要承担：
- 进度条
- 当前进度数字
- 当前任务节奏感

顶部不再承担：
- 强操作按钮
- 顶部角落关闭图标
- 模糊的中间退出入口

### 3. 弱化退出，但不能隐藏退出
退出必须满足：
- 家长能一眼找到
- 儿童不容易误触
- 不和微信胶囊冲突
- 不破坏沉浸体验

### 4. 主按钮永远是页面第一行动点
每个页面必须有明确的单一主行动点：
- 学习页：继续学习
- 测试页：继续答题 / 继续下一步
- 复习页：认识 / 不认识
- 完成页：下一单元 / 返回首页

## 页面级整改方案

### 一、学习页 `src/subpkg-learning/learn/index.vue`
#### 页面角色
核心学习页，沉浸感最强，主任务是“认字 + 理解 + 继续”。

#### 整改方向
- 顶部使用统一的沉浸式学习头部
- 第一行只放进度条和步数
- 第二行左侧放弱化退出入口 `← 退出学习`
- 不使用右上角关闭按钮
- 不把退出入口放在页面顶部中间

#### 内容区层级
主视觉层级应为：
1. 当前字 / 图 / 发音
2. 学习解释卡片
3. 组词辅助信息
4. 主按钮 `我记住啦` / `继续学习`

#### 原型
```text
┌──────────────────────────────────┐
│ [███████████──────]        3/16  │
│ ← 退出学习                        │
├──────────────────────────────────┤
│              [字源图]            │
│                耳                │
│               ěr                 │
│         [🔊 点击听读音]           │
│ ┌──────────────────────────────┐ │
│ │ 象形   字源演变               │ │
│ │ 像人的耳朵形状               │ │
│ └──────────────────────────────┘ │
│ 常用组词                         │
│ [耳朵]   [耳机]   [木耳]         │
│        [我记住啦 / 继续学习]     │
└──────────────────────────────────┘
```

#### 视觉建议
- 顶部退出入口：暖灰色、26~28rpx、左对齐
- 当前字字号继续保持最大层级
- 拼音与发音入口合并为同一认知组
- 学习知识卡增加更清晰卡片感与留白
- 主按钮继续大圆角、暖色高亮

---

### 二、单元测试页 `src/subpkg-learning/learn/unit-test.vue`
#### 页面角色
任务型答题页，强调题目推进和完成感。

#### 整改方向
- 顶部第一行显示进度条 + 题数
- 第二行左侧放弱化退出入口 `← 退出测试`
- 不再使用右上角 `✕`
- 退出入口比学习页稍微清晰一些，但仍然不是主焦点

#### 内容区层级
主视觉层级应为：
1. 题型标签
2. 题干核心内容
3. 选项区
4. 反馈条 / 下一步

#### 原型
```text
┌──────────────────────────────────┐
│ [███████─────────]         2/8   │
│ ← 退出测试                        │
├──────────────────────────────────┤
│            [看拼音选字]           │
│                ěr                │
│         请选择正确的汉字          │
│    [耳]      [目]      [手]      │
│    [口]                           │
└──────────────────────────────────┘
```

#### 视觉建议
- 题型标签保持小标签但更规整
- 反馈层保留，但颜色更友好，避免强烈警示感
- 退出入口颜色使用中性暖灰，不用警示色

---

### 三、复习页 `src/subpkg-learning/review/index.vue`
#### 页面角色
轻量复习页，目标是快速翻卡、快速判断“认识 / 不认识”。

#### 整改方向
- 顶部统一为进度条 + 左侧退出入口 `← 退出复习`
- 不再放右上角 `✕`
- 顶部结构与学习页保持同规则，但退出入口可比学习页略清楚一点

#### 内容区层级
主视觉层级应为：
1. 闪卡
2. 翻面信息
3. 底部双按钮（认识 / 不认识）
4. 退出入口

#### 原型
```text
┌──────────────────────────────────┐
│ [██████████──────]        4/10   │
│ ← 退出复习                        │
├──────────────────────────────────┤
│            [闪卡区域]            │
│                耳                │
│          点击翻转查看读音         │
│       [不认识]      [认识]        │
└──────────────────────────────────┘
```

#### 视觉建议
- `认识` 按钮维持积极色
- `不认识` 按钮避免使用威胁感强的红色
- 闪卡区域继续保持大面积留白和节奏感

---

### 四、单元完成页 `src/subpkg-learning/learn/unit-complete.vue`
#### 页面角色
结果与分发页，目标是给孩子成就感，并给家长明确下一步。

#### 整改方向
- 继续保留顶部弱化回首页入口
- 中部继续强调星级、标题、统计数据
- 底部主按钮继续作为第一行动点
- 与学习页统一语言风格，但不改成沉浸式学习头部

#### 原型
```text
┌──────────────────────────────────┐
│ ← 回首页                          │
│              ⭐⭐⭐                │
│            单元完成！             │
│            第一单元               │
│   学会汉字   正确率   用时   连续天数 │
│      8个      90%    3分   5天      │
│      [本单元学会的汉字墙]          │
│        [生成海报] [分享好友]       │
│          [下一单元 / 返回首页]     │
└──────────────────────────────────┘
```

#### 视觉建议
- 奖励感强于导航感
- 顶部回首页入口继续弱化
- 不让顶部操作抢走成就反馈的视觉中心

## 统一头部规范
### 结构规范
所有沉浸式任务页统一采用：
1. 顶部进度条 + 步数
2. 下方左侧弱化退出入口
3. 再进入主内容区

### 样式建议
- 顶部左右内边距：`32rpx`
- safe area 后的额外顶部间距：`20~24rpx`
- 进度条与退出入口间距：`12~16rpx`
- 退出入口与内容区间距：`20~24rpx`
- 退出入口字号：`26~28rpx`
- 点击热区高度：至少 `56rpx`

### 文案建议
- 学习页：`← 退出学习`
- 测试页：`← 退出测试`
- 复习页：`← 退出复习`
- 完成页：`← 回首页`

## 统一视觉规范
### 色彩
- 学习主色：暖黄 / 橙黄
- 复习主色：浅蓝 / 蓝灰
- 退出入口：低饱和中灰 / 暖灰 / 蓝灰
- 主按钮：高亮、正向、可点击感强

### 圆角与卡片
- 主卡片、知识卡、闪卡、结果卡统一更圆润
- 儿童内容区保留更多留白
- 避免细密信息堆积在顶部

### 按钮
- 主按钮：明显、圆角大、文案简短
- 次按钮：弱化但点击区不能小
- 文字按钮优于抽象 icon，尤其在家长辅助操作区

## 优先级建议
### P0（立即整改）
- 学习页顶部结构与退出入口位置重构
- 测试页顶部结构与退出入口位置重构
- 复习页顶部结构与退出入口位置重构
- 三页统一沉浸式头部规范

### P1（紧随其后）
- 学习页内容区层级优化
- 测试页题型标签 / 题干 / 选项层级优化
- 复习页闪卡和底部按钮间距优化
- 单元完成页与前序学习流视觉语言统一

### P2（后续增强）
- 学习页增加当前环节提示
- 完成页奖励感更游戏化
- 抽象出整套学习流设计 token

## 验收标准
- 学习页、测试页、复习页的退出入口均不再位于胶囊冲突区
- 儿童第一眼看到的是学习任务，而不是退出入口
- 家长能够快速识别退出入口
- 顶部进度信息清晰可读
- 主按钮始终是当前页面最清晰的行动点
- 单元完成页继续以成就反馈和下一步为主，不被导航打断

## 结论
从专业儿童线上教育产品经理和专业 UI 设计师角度判断，学习相关页面的整改重点，不应继续停留在“按钮能否显示”，而应升级为“学习流页面是否真正适合儿童和家长共同使用”。

本次推荐的最终方案是：
- 学习页、测试页、复习页统一为“顶部进度 + 左侧弱化退出入口”的沉浸式头部模式
- 页面主视觉严格围绕学习任务组织
- 完成页继续走奖励与下一步导向
- 整体形成一套儿童友好的学习流页面规范
