# 剩余二级页面返回入口排查与 loot 页面导航设计

- 日期：2026-03-30
- 范围：`shizi-frontend` 剩余二级页面返回入口排查
- 状态：已确认，待进入实现规划

## 背景

在协议页、隐私页和 `unit-complete` 页面完成导航优化后，项目中仍有若干非 tabbar 页面尚未重新评估返回入口策略。

本轮目标不是重做所有页面导航，而是聚焦“真正缺少明确返回入口、且会明显影响用户理解与退出效率”的页面，优先补齐高价值缺口。

结合当前代码结构，剩余重点页面包括：
- [src/pages/loot/index.vue](../../../../shizi-frontend/src/pages/loot/index.vue)
- [src/pages/login/index.vue](../../../../shizi-frontend/src/pages/login/index.vue)
- [src/pages/onboarding/index.vue](../../../../shizi-frontend/src/pages/onboarding/index.vue)

## 设计目标

1. 只处理真正缺少返回入口且影响明显的页面，避免过度设计。
2. 延续前一轮已建立的导航分型原则，而不是重新推翻整套规则。
3. 在儿童教育产品语境下，区分“儿童主操作页”和“家长/辅助浏览页”的导航设计。
4. 为下一步实现提供明确优先级，而不是一次性扩散到所有页面。

## 非目标

1. 不在本轮统一重构所有非 tabbar 页头部。
2. 不在本轮重做登录转化流程。
3. 不在本轮决定 onboarding 的完整引导策略。
4. 不在本轮修改学习流页面的关闭/退出逻辑。

## 页面排查结论

### 1. loot 页面

页面：
- [src/pages/loot/index.vue](../../../../shizi-frontend/src/pages/loot/index.vue)

结论：
- 应处理，且为本轮优先级最高页面。

页面定位：
- `loot` 应归类为**标准功能型二级页**。
- 它不是一级 tab 页面，不是学习任务流，不是登录入口页，也不是结果结算页。
- 用户进入该页的核心心智是“浏览内容、点开详情、再离开页面”。

问题判断：
- 当前页面具有自定义 hero 区域，但缺少清晰的页面级返回入口。
- 从“我的”页进入后，用户缺乏页面内的稳定退出路径。
- 该问题对家长用户和低频用户都较明显，属于“真实缺口”而非细节优化。

设计原则：
- `loot` 需要明确返回语义，而不是关闭语义。
- 不使用学习流式“退出”逻辑，也不需要确认弹窗。
- 应纳入前面已建立的二级页导航规则，但允许保留本页视觉特色。

### 2. login 页面

页面：
- [src/pages/login/index.vue](../../../../shizi-frontend/src/pages/login/index.vue)

结论：
- 当前不属于“明显缺返回入口”的页面。
- 本轮不作为优先处理对象。

原因：
- 页面已有“暂不登录”入口。
- 当前 `goBack()` 已具备“优先返回上一页，兜底回首页”的逻辑。
- 因此它并非无法退出，只是退出入口以底部弱文案形式存在。

产品判断：
- 这是“可优化体验”，不是“明显缺陷”。
- 若后续要做登录页转化与退出路径优化，可作为单独专题处理。
- 本轮不建议和 `loot` 混合落地，以免范围扩散。

### 3. onboarding 页面

页面：
- [src/pages/onboarding/index.vue](../../../../shizi-frontend/src/pages/onboarding/index.vue)

结论：
- 本轮不直接处理。

原因：
- 它是否需要退出入口，不是单纯的返回按钮问题，而是引导策略问题。
- 如果产品目标是强引导，则可以不提供退出。
- 如果产品目标是可跳过引导，则需要设计“跳过 / 稍后设置 / 关闭”等入口。

产品判断：
- `onboarding` 不应直接套用普通二级页规则。
- 它应在未来作为“引导策略”专题单独讨论。

## loot 页面导航设计

## 页面定位

`loot` 页面属于标准功能型二级页。

其职责是：
- 展示汉字图鉴
- 展示收集进度
- 提供复习入口
- 提供汉字详情与重播能力

它不承载连续任务，不存在“退出会中断流程”的问题，因此应采用标准返回语义。

## 交互定位

`loot` 的导航分工如下：

- **主内容区**：承担浏览、查看、进入详情/复习等功能。
- **顶部返回入口**：承担离开当前页面、回到来源页的作用。

这里的返回入口不需要承担主任务操作，因此左上角入口是合理的。

在儿童教育产品语境下，这类页更偏向“家长辅助浏览页”而不是“儿童主任务操作页”，因此不需要像学习页或结果页那样把主要操作放在底部大按钮中。

## 推荐方案

### 方案结论

- 保留当前 hero 区域视觉。
- 不新增完整标准 navbar。
- 在 hero 左上角增加一个轻量返回入口。

### 为什么不加完整 navbar

原因：
- 页面顶部已经有“汉字图鉴”标题、收集进度和进度条。
- 如果额外叠加完整 navbar，会导致头部层级过重。
- 会削弱当前页面已有的识别度与内容氛围。

### 为什么要嵌在 hero 里

原因：
- 更符合页面当前结构。
- 可以保持视觉完整性。
- 能同时满足“有明确返回入口”和“不过度模板化”的目标。

## loot 返回入口的具体设计

### 位置

- 放置在 hero 区域左上角。
- 位于安全区内，避免贴边。

### 视觉形式

- 轻量箭头按钮，允许配合“返回”弱文案。
- 使用半透明浅色底或浅色描边，保证在 hero 背景上可见。
- 视觉权重低于页面标题与主内容，不抢占注意力。

### 交互语义

- 视觉语义：返回。
- 不使用“关闭”或“退出”文案。
- 不增加确认弹窗。

### 行为规则

1. 优先返回上一页。
2. 若当前页没有历史栈，则兜底回“我的”页。
3. 不允许出现点按后无反应的情况。

兜底目标：
- `/pages/me/index`

原因：
- 当前 `loot` 的主要入口来自“我的”页。
- 与用户心智一致。
- 比直接回首页更自然。

## login / onboarding 的处理策略

### login

当前策略：
- 保持现状。
- 暂不新增顶部返回或关闭入口。

后续可优化方向：
- 若未来发现家长用户在登录页频繁寻找顶部退出，可补一个弱化关闭入口。
- 但这不属于本轮高优先级任务。

### onboarding

当前策略：
- 暂不处理。

后续需单独确认的问题：
- 该页是否允许跳过。
- 若允许跳过，应回首页、返回登录页，还是保留当前选择后进入首页。
- 跳过是顶部入口还是底部辅助文案。

这些问题超出本轮“缺失返回入口排查”范围。

## 最终优先级

### 本轮应立即处理

1. [src/pages/loot/index.vue](../../../../shizi-frontend/src/pages/loot/index.vue)

### 本轮不急，但未来可优化

2. [src/pages/login/index.vue](../../../../shizi-frontend/src/pages/login/index.vue)

### 本轮不建议直接处理

3. [src/pages/onboarding/index.vue](../../../../shizi-frontend/src/pages/onboarding/index.vue)

## 最终结论

1. 在剩余页面中，真正缺少明确返回入口且影响明显的页面是 `loot`。
2. `loot` 应被归类为标准功能型二级页。
3. `loot` 应保留当前 hero 视觉，并在 hero 左上角增加轻量返回入口。
4. 该入口应采用“优先返回上一页，兜底回我的页”的行为规则。
5. `login` 当前可用但可优化，不是本轮优先项。
6. `onboarding` 是否需要退出入口属于引导策略问题，应单独讨论。
