# 沉浸式学习页退出入口布局重构设计

## 背景
前面已经确认，协议页和隐私页通过共享标准 navbar 已经修复完成；但学习页、单元测试页、复习页中的右上角关闭按钮仍然会被微信自带胶囊遮挡。

用户进一步确认后，希望从布局优化角度解决，而不是继续把关闭按钮放在页面右上角。这个判断是合理的：右上角是微信小程序里与胶囊冲突最强的区域之一，继续围绕这个位置做精确避让，虽然理论上可做，但长期稳定性和维护收益都不高。

## 目标
- 让沉浸式学习页的退出入口稳定可见
- 不再把退出入口放在右上角胶囊冲突区
- 保持沉浸式学习页的产品语义，不改成标准 navbar
- 保持现有退出确认和 fallback 逻辑不变

## 方案选择
本次采用“布局重构而非继续右上避让”的方案：
- 保留沉浸式页面现有进度区结构
- 将退出入口移入顶部安全内容区
- 用弱化的辅助操作形式呈现退出能力

不采用继续把 `✕` 放在右上角并反复避让胶囊的方案，因为这会持续依赖端上特殊布局细节，稳定性不足；也不采用改成标准 navbar 的方案，因为这会破坏之前已经确定的沉浸式页面体验。

## 设计方案
### 1. 三个沉浸式页面统一调整退出入口布局
本次统一调整以下页面：
- `src/subpkg-learning/learn/index.vue`
- `src/subpkg-learning/learn/unit-test.vue`
- `src/subpkg-learning/review/index.vue`

这三页都保留现有顶部进度信息，但不再在右上角放单独关闭按钮。

### 2. 退出入口移动到顶部安全内容区内
退出入口改成顶部安全内容区中的弱化操作，不再贴住右上角。

布局方向：
- 第一层继续承载顶部安全距离与进度信息
- 退出入口进入同一安全区内的可控布局区域
- 可以与进度条形成上下结构，或在进度条左侧形成辅助入口

本次推荐统一采用：
- 顶部先显示进度条与进度数字
- 在其下增加一行弱化退出入口（例如“退出学习”“退出测试”“退出复习”）

这样做的好处是：
- 完全避开胶囊冲突区
- 退出入口稳定可见
- 比角落小 `✕` 更容易命中
- 对儿童教育产品更友好，弱化但清晰

### 3. 保持沉浸式页面语义
本次只调整退出入口的布局，不改变页面本身的沉浸式语义：
- 不改成标准 navbar
- 不引入左返回箭头
- 不改变主内容层级
- 不改变继续学习 / 答题 / 复习的主路径

退出入口依然是辅助性的，不应抢占主视觉焦点。

### 4. 交互逻辑不变
以下逻辑保持原样：
- 学习页退出前的确认弹窗
- 测试页退出前的确认弹窗
- 复习页退出前的确认弹窗
- 已有的 fallback 返回逻辑

本次只改“入口位置与布局”，不改“点击后发生什么”。

### 5. 视觉与文案策略
退出入口建议从单字符 `✕` 改为更明确但弱化的文本型入口，例如：
- 学习页：`退出学习`
- 测试页：`退出测试`
- 复习页：`退出复习`

视觉建议：
- 小字号
- 中低对比度颜色
- 文本按钮或轻量描边按钮
- 与顶部进度区保持一定间距

这样既提升可理解性，也避免角落图标在儿童场景下过于隐蔽。

## 验收标准
- 学习页可稳定看到退出入口
- 单元测试页可稳定看到退出入口
- 复习页可稳定看到退出入口
- 退出入口不再被微信胶囊遮挡
- 顶部进度条和步数信息仍清晰可读
- 页面仍保持沉浸式学习体验
- 退出确认与 fallback 逻辑保持不变

## 影响范围
- `src/subpkg-learning/learn/index.vue`
- `src/subpkg-learning/learn/unit-test.vue`
- `src/subpkg-learning/review/index.vue`

如有必要，可继续复用已有的 `src/utils/navbar.ts` 作为顶部安全区能力来源，但本次重点不再是右上精确避让，而是退出入口布局重构。

## 不在本次范围内
以下不在本次整改范围内：
- 协议页 / 隐私页标准 navbar
- login 页
- loot 页
- onboarding 页
- unit-complete 页
- 其他非沉浸式学习流程页面

## 测试建议
- 在微信开发者工具中分别进入学习页、单元测试页、复习页
- 确认退出入口稳定可见
- 确认入口不再落入右上胶囊区域
- 点击退出入口，确认弹窗与既有跳转逻辑不变
- 确认顶部进度条与进度数字布局仍然清晰
