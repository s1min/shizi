# Unit List 页面 Modern 主题精修设计

日期：2026-04-05
页面：`src/pages/unit-list/index.vue`
目标：在已完成的 unit-list 页面 modern 主题基础上，删除 cream 对比主题，并从资深 UI 设计工程师与资深前端工程师视角做一轮现代精修，让页面更克制、更高级、更适合线上儿童教育产品的稳定发布态。

## 1. 背景

当前 unit-list 页面已经完成一次现代化改版，核心收益包括：

- 顶部区域改为固定导航区；
- 字库摘要卡完成信息整合；
- 图例移除；
- 单元卡已具备现代化 badge、按钮与底部操作结构；
- 已支持 cream / modern 两种视觉对比。

用户已经明确选择 **modern** 作为最终方向，因此本轮不再保留双主题对比逻辑，而是将页面全面收敛为一套稳定的 modern 风格，并对首屏、摘要卡、单元卡、按钮与列表节奏进行精修。

## 2. 现状问题分析

### 2.1 顶部区域仍偏柔

虽然当前顶部已经有固定渐变背景，但 navbar 渐变、tabs 容器、页面浅背景和摘要卡高光叠加后，首屏上半区整体偏“糊”，缺少清晰的视觉分层。

结果是：
- 顶部导航层和正文主视觉层的边界不够明确；
- tabs 的导航属性不够纯粹；
- 首屏焦点容易在多个浅色面之间分散。

### 2.2 字库摘要卡“大而散”

`LibrarySummaryCard` 当前结构已经正确，但内部视觉重心仍不够集中：
- 标题、说明、切换按钮、4 个统计块都在争夺权重；
- “切换字库”按钮尺寸略大，工具属性不足；
- 统计块仍有一点“四块拼接卡片”的观感。

### 2.3 单元卡内容主次仍可优化

当前单元卡的核心问题不是结构，而是信息与装饰的权重分配：
- 右上角 badge 虽然比旧版更现代，但仍略像“贴上去”的状态块；
- 绿色状态 badge 视觉权重偏高，容易先看到状态再看到标题；
- progress-strip 的面积和存在感仍略强。

### 2.4 按钮区略显厚重

底部 `复习 / 重新测试` 按钮方向是对的，但仍有两个问题：
- 主按钮蓝色偏跳，稍有“促销按钮感”；
- 次按钮偏厚，和主按钮放在一起时缺少更克制的层次差。

### 2.5 列表连续阅读节奏还不够成熟

连续卡片列表在当前版本下略显“鼓”：
- 单卡高度稍高；
- 卡片之间的节奏略松；
- 标题、正文、进度、footer 的垂直距离都可以再压一档。

这会让长列表场景下的滚动体验稍显松散。

## 3. 总体设计决策

本轮采用“**删掉 cream，modern 单主题精修**”策略。

即：
- 删除 `cream` 主题变量、分支和相关样式；
- 保留 `modern` 主题作为唯一页面视觉；
- 不改变现有业务结构与交互流程；
- 只对视觉层与节奏层进行稳定化打磨。

## 4. 最终视觉收敛原则

### 4.1 背景只做氛围，不做主角

- 页面背景保留极淡冷白与浅蓝氛围；
- 去掉 warm / cream 的残留色调；
- 顶部背景仍允许轻微渐变，但正文区要更干净。

目标：让背景只承担“现代、轻盈”的气氛作用，而不是构成视觉负担。

### 4.2 首屏只有一个主视觉中心

首屏主视觉中心明确收敛到：
**字库标题 + 统计摘要卡**。

因此：
- navbar 和 tabs 只承担导航作用，不应抢正文；
- “切换字库”属于工具动作，应低于字库标题权重；
- 首屏信息必须先让用户看到当前学习字库，再看到关键统计，再进入列表内容。

### 4.3 单元卡强调内容，弱化装饰

单元卡的主阅读路径收敛为：
1. 单元标题
2. lesson / 字词内容
3. 状态进度
4. 操作按钮

因此需要：
- badge 更内嵌、更轻；
- progress-strip 更淡；
- 卡片渐变、阴影和色块都再收一档；
- 卡片不要因为装饰而压过内容本身。

### 4.4 按钮系统回归产品化

按钮语义明确分层：
- 工具按钮：切换字库
- 次按钮：复习
- 主按钮：重新测试

视觉要求：
- 主按钮有推进感，但不过于跳；
- 次按钮更轻更薄；
- 工具按钮显著小于主 CTA；
- 三类按钮风格统一，但权重清晰。

### 4.5 列表节奏收紧一档

- 卡片高度略降；
- 卡片内部模块间距略缩；
- 卡片间距略缩；
- 分组标题与卡片关系更清晰。

目标是让页面更像成熟的内容列表，而不是大面积展示卡集合。

## 5. 具体改动方案

## 5.1 页面根层与主题收敛

### 当前问题
`index.vue` 仍保留：
- `previewTheme`
- `pageThemeClass`
- cream 默认变量
- `.theme-modern` 覆盖逻辑

### 改动决策
- 删除 `previewTheme` 与 `pageThemeClass`。
- 页面根节点仅保留 `.unit-page`。
- 直接把 modern 的变量值提升为默认变量。
- 删除 `.theme-modern` 相关覆盖块。

### 结果
- 代码更干净；
- 页面主题不再有临时态；
- 后续不会再出现“页面默认不是最终主题”的混乱。

## 5.2 顶部 fixed 区精修

### 目标
让顶部成为一个更纯粹的导航层，而不是柔和装饰层。

### 调整方向
- navbar 背景改为更偏冷白的低对比渐变；
- 降低顶部背景的奶油色与暖调；
- 增加更细腻的底部分隔线或轻阴影；
- tabs 所在区域对比略增强，使其从背景中更清晰地浮现。

### 预期效果
- 顶部与正文分层更清楚；
- 首屏不会显得“糊成一片”；
- 页面更像成熟产品，而不是视觉展示稿。

## 5.3 字库摘要卡精修

### 目标
把视觉重心收回到“字库名 + 数据摘要”。

### 调整方向
- 卡片整体垂直高度收紧；
- 标题字重和颜色维持稳定，但说明文案继续弱化；
- “切换字库”按钮缩小一档，减小高度、内边距与背景浓度；
- 4 个统计块保留，但进一步统一为更轻的信息块；
- 统计块背景更淡，减少独立小卡片感。

### 预期效果
- 首屏主信息更集中；
- 按钮不再抢标题；
- 摘要卡看起来更精致，不松散。

## 5.4 单元卡精修

### 目标
让单元卡更像“内容卡”，不是“状态卡”。

### 调整方向
- badge 高度和内边距缩小一档；
- badge 圆角切面更规整，与卡片圆角关系更统一；
- badge 饱和度降低一点，避免压过标题；
- 卡片上半区的标题与 badge 顶对齐关系优化；
- progress-strip 背景更淡、边界更轻；
- 卡片本身渐变与阴影再减弱。

### 预期效果
- 用户先读到单元内容，而不是先看到状态块；
- 卡片更高级，更符合 modern 最终版定位。

## 5.5 Footer 与按钮区精修

### 目标
让卡片底部更稳、更现代。

### 调整方向
- stars 与按钮垂直基线更统一；
- stars 保留奖励感，但减少松散感；
- 次按钮 `复习` 更轻、更薄、更工具化；
- 主按钮 `重新测试` 蓝色降低一点亮度，减少“电商 CTA”感；
- 两个按钮宽度比例更协调。

### 预期效果
- footer 看起来更成熟；
- 主次操作更清晰；
- 按钮既有可点性，又不显得笨重。

## 5.6 分组与列表节奏精修

### 目标
提高长列表滚动体验。

### 调整方向
- 分组标题与首张卡的间距略收；
- 卡片之间间距略缩；
- 卡片内部标题、正文、状态条、footer 之间间距压一档；
- 保持舒适，但减少拖沓感。

### 预期效果
- 页面更紧凑；
- 连续阅读更轻松；
- 首屏可见信息更多。

## 6. 实现范围

涉及文件：
- `src/pages/unit-list/index.vue`
- `src/pages/unit-list/components/LibrarySummaryCard.vue`
- `src/pages/unit-list/components/UnitTaskCard.vue`
- `src/pages/unit-list/components/UnitStageSection.vue`

不涉及：
- `useUnitListPage.ts` 业务逻辑
- 数据结构与 view-model 字段
- 路由和跳转行为
- 错字流程逻辑

## 7. 验收标准

本轮精修完成后，应满足：

1. 页面不再保留 cream 主题及切换逻辑。
2. `modern` 成为唯一主题实现。
3. 顶部 fixed 区比当前更干净，分层更明确。
4. 字库摘要卡更紧凑，“切换字库”按钮更工具化。
5. 统计块更轻，减少四宫格小卡片感。
6. 单元卡 badge 更融合，不抢标题注意力。
7. progress-strip 更淡，卡片整体更克制。
8. `复习 / 重新测试` 按钮更现代、更高级，主次关系更清晰。
9. 列表整体阅读节奏更紧凑、更成熟。
10. 页面整体保留儿童教育产品的友好感，但显著提升现代感与上线完成度。

## 8. 非目标

以下内容不在本轮范围：
- 不重做页面结构；
- 不更换组件体系；
- 不修改数据项数量与业务文案；
- 不引入新的交互动效；
- 不做新的双主题对比能力。

## 9. 结论

本轮属于 **modern 主题定稿前的精修收敛**，核心不是“再做一次设计”，而是：
- 删掉对比实验态；
- 收紧页面层级；
- 让现代感真正稳定下来；
- 把当前页面从“已经不错”推进到“可作为稳定线上版本”。
