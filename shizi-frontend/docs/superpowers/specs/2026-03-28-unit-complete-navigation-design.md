# unit-complete 结果页导航设计

- 日期：2026-03-28
- 范围：`shizi-frontend` `unit-complete` 页面导航体验
- 状态：已确认，待进入实现规划

## 背景

在二级页面导航规范完成后，`unit-complete` 仍需要单独判断其导航归类。该页面位于学习流程结束后，当前页面结构以星级、学习统计、识字成果、海报分享和“下一单元 / 返回首页”主按钮为核心，明显区别于协议、隐私这类标准内容页，也区别于学习中、测试中、复习中这类强沉浸任务执行页。

因此，`unit-complete` 不能直接套用标准 navbar，也不应继续完全沿用“流程进行中页面”的退出逻辑，而应基于儿童教育产品的结果页心智单独设计。

## 设计目标

1. 保持单元完成页的庆祝感、结果感和下一步引导能力。
2. 让儿童用户优先通过大按钮完成下一步操作。
3. 为家长或有明确退出需求的用户提供轻量、稳定的辅助出口。
4. 避免将结果页错误地设计成“返回上一页”的普通二级页。

## 非目标

1. 不将 `unit-complete` 纳入标准 navbar 体系。
2. 不重构页面主体内容、结果卡片、海报分享或底部 CTA 结构。
3. 不引入历史栈判断或复杂回退逻辑。
4. 不把左上角入口设计成主要交互。

## 页面定位

`unit-complete` 的产品定位应为：

### 结果分流页

含义：
- 学习流程已经完成。
- 当前页面的职责是“总结成果 + 鼓励用户 + 引导下一步”。
- 用户此时最需要的是明确的下一步去向，而不是回到上一流程节点。

这意味着该页面既不是：
- 标准内容型二级页
- 强沉浸流程执行页

而是应作为单独的结果分流页进行处理。

## 导航层级

### 1. 主导航

主导航由页面底部大按钮承担：
- `下一单元`
- `返回首页`

设计要求：
- 继续保持当前页面的主操作优先级。
- 这两个大按钮是儿童用户最容易理解、最容易点击的导航路径。
- 页面视觉重心应继续围绕主结果内容和底部 CTA 展开。

### 2. 辅助导航

页面左上角增加一个轻量辅助入口：
- 使用图标 + 短文案
- 行为固定为回首页
- 仅作为次级导航存在，不参与主流程引导

设计要求：
- 视觉层级弱于底部主按钮。
- 点击热区足够大，便于家长代操作。
- 不使用完整标准 navbar 样式。
- 不强调“返回上一页”。

## 为什么不采用标准返回按钮

### 不适合使用“返回上一页”

原因：
1. `unit-complete` 前一页通常是测试页或学习流程页，语义上已完成，不适合作为结果页的默认回退目标。
2. 返回到上一页可能带来流程状态混乱，例如回到已结束流程。
3. 儿童和家长更需要稳定、可预期的导航结果，而不是依赖 history 栈。

结论：
- 不应将左上角入口设计为“返回上一页”。
- 不应依赖 `navigateBack` 作为该页面的默认离开方式。

### 不适合接入完整 navbar

原因：
1. 标准 navbar 会削弱当前页面的庆祝感和结果页仪式感。
2. 会将结果页错误包装成普通二级信息页。
3. 小尺寸顶部返回按钮不适合作为儿童产品中的主导航方式。

结论：
- 顶部只补轻量辅助入口，不接入协议/隐私页那种标准 navbar。

## 左上角入口的具体设计

### 推荐形态

推荐使用：
- 左箭头/主页图标 + `首页`

而不是：
- 单独的返回箭头
- 单独的关闭 `×`

原因：
1. “首页”比“关闭”更明确，用户可以预判点击结果。
2. 图标 + 文案组合比纯图标更适合儿童和家长理解。
3. 该入口表达的是“离开结果页并回到主入口”，而不是“回到上一步”。

### 视觉规范

- 放置于左上角安全区域内。
- 颜色、字号、对比度弱于底部主 CTA。
- 点击热区需大于纯图标尺寸，避免误触困难。
- 不应形成完整标题栏压迫页面主视觉。

## 交互行为

### 顶部辅助入口

行为：
- 固定回首页。
- 使用稳定入口跳转，例如 `switchTab('/pages/home/index')` 或同等能力。
- 不进行 history 判断。
- 不回退到上一页。

### 底部主按钮

继续保留现有策略：
- 有下一单元时，主按钮引导进入下一单元。
- 无下一单元时，主按钮引导返回首页。

### 是否需要确认弹窗

结论：
- 不需要确认弹窗。

原因：
1. 当前页面已经完成单元流程，不存在中断损失。
2. 退出结果页不会丢失学习进度。
3. 结果页应该是轻量离开的，不应套用学习中/测试中的退出确认机制。

## 与儿童教育产品心智的匹配

### 适合儿童的部分

- 主操作继续由大按钮承担，点击目标大、文案明确。
- 页面情绪和奖励感保持不变。
- 下一步路径清晰，不让儿童依赖小型顶部返回按钮完成主要操作。

### 适合家长的部分

- 左上角补充了稳定、直观的辅助出口。
- 行为固定可预期，不需要理解页面栈。
- “首页”文案比“关闭”或“返回”更容易快速判断。

## 当前代码的落地边界

### 保留不变

以下内容保持现状：
- 星级展示
- 成绩统计卡
- 学会的字墙
- 海报生成与分享能力
- 底部主 CTA 的整体结构与优先级

对应页面：
- [src/subpkg-learning/learn/unit-complete.vue](../../../../shizi-frontend/src/subpkg-learning/learn/unit-complete.vue)

### 需要新增

新增内容仅限：
- 左上角轻量“首页”辅助入口
- 固定回首页行为
- 适配结果页视觉层级的轻量样式

### 不做的事情

- 不新增完整 navbar
- 不新增页面标题栏
- 不接入标准返回 helper 的 history 回退分支
- 不弹确认框
- 不改变底部主按钮的主路径职责

## 最终结论

1. `unit-complete` 应定义为结果分流页，而非普通二级页。
2. 页面主导航继续由底部大按钮承担。
3. 页面左上角补一个轻量辅助入口即可。
4. 辅助入口应固定回首页，不应返回上一页。
5. 不接入标准 navbar，不弹确认框。
6. 该方案更符合儿童教育产品“主路径明确、辅助路径稳定、结果可预期”的导航原则。
