# 登录页弱化返回入口设计

- 日期：2026-03-30
- 范围：`shizi-frontend` 登录页导航优化
- 状态：已确认，待进入实现规划

## 背景

当前登录页 [src/pages/login/index.vue](../../../../shizi-frontend/src/pages/login/index.vue) 已具备底部“暂不登录”入口，并且 `goBack()` 逻辑已经支持“优先返回上一页，兜底回首页”。因此它并不属于“无法退出”的缺陷页面。

但从移动端通用交互习惯和家长用户直觉来看，用户进入登录页后，往往会先扫左上角是否存在退出路径。仅依赖底部弱文案，会使退出入口虽然可用，但不够顺手。

因此本次优化的目标不是增强退出存在感，而是在不影响登录转化的前提下，补充一个更符合直觉的弱化顶部返回入口。

## 设计目标

1. 为登录页补充一个符合移动端习惯的弱化顶部返回入口。
2. 保持登录页转化页属性，不让返回入口过度抢占注意力。
3. 保留当前“暂不登录”文案入口，使导航直觉和策略表达同时成立。
4. 不改动现有登录、体验模式、跳过登录的核心流程逻辑。

## 非目标

1. 不重做登录页版式。
2. 不改造登录转化文案。
3. 不重做登录成功后的跳转链路。
4. 不将登录页改造成标准 navbar 页面。

## 页面定位

登录页属于**转化页 / 流程入口页**，而不是普通标准二级页。

它的主要职责是：
- 引导用户完成登录
- 解释登录收益
- 提供可跳过的替代路径

因此，登录页不适合像协议页、隐私页那样使用完整标准 navbar；否则会削弱页面的沉浸感和转化聚焦。

## 推荐方案

### 方案结论

- 在左上安全区增加一个**弱化返回箭头**。
- 仅使用箭头图标，不配返回文案。
- 保持当前底部“暂不登录”入口不变。
- 不增加完整 navbar，不增加确认弹窗。

## 为什么不做完整 navbar

原因：
- 登录页本身是完整的转化页结构。
- 如果增加完整 navbar，会让页面更像普通功能页，而不是登录引导页。
- 会抬高返回入口权重，不符合“希望用户登录，但允许退出”的产品目标。

## 为什么只放箭头不放文案

原因：
- 文案会显著增加返回入口的视觉存在感。
- 登录页主目标仍然是登录或体验，不应让“返回”与主 CTA 竞争。
- 单纯箭头更适合作为辅助出口。

因此，推荐：
- 只保留图标
- 不写“返回”
- 不写“关闭”
- 不写“暂不登录”

## UI 设计

### 位置

- 放置在页面左上安全区。
- 不额外创建整条顶栏。
- 不与 logo 区混成一个标准头部结构。

### 视觉形式

- 单独的箭头型按钮。
- 颜色采用浅灰、柔和深色或半透明深色。
- 点击热区足够，但视觉本体尽量轻。
- 整体存在感应弱于主登录按钮、体验按钮和主 logo。

### 视觉权重要求

返回箭头的权重应：
- 低于微信一键登录 / 开始体验按钮
- 低于主 logo 与页面主信息
- 高于背景装饰

即：
- 看得见
- 找得到
- 但不成为主视觉焦点

## 交互逻辑

### 点击行为

沿用当前 `goBack()` 逻辑：
1. 若有上一页，则返回上一页。
2. 若没有上一页，则兜底回首页。

兜底目标：
- `/pages/home/index`

### 不需要的交互

- 不增加确认弹窗。
- 不新增关闭文案。
- 不区分“返回”和“暂不登录”的逻辑分支。

原因：
- 登录页退出不涉及进度损失。
- 当前已有稳定返回逻辑。
- 增加额外分支会提高复杂度，但收益很小。

## 与“暂不登录”的关系

### 设计结论

- 左上箭头保留。
- 底部“暂不登录”也保留。

### 角色分工

左上箭头：
- 承担移动端直觉型退出入口。
- 满足用户对顶部返回的预期。

底部“暂不登录”：
- 承担明确文案表达。
- 向用户说明“不登录也可以继续使用”。

### 为什么两者不冲突

虽然两者都可能最终触发 `goBack()`，但承担的是不同心智：
- 箭头解决“怎么退出这个页面”
- “暂不登录”解决“我能不能不登录继续用”

因此两者应并存，而不是互相替代。

## 产品结论

登录页当前不是“严重缺陷页”，但值得做一个小而稳的优化。

这是一个：
- 低风险
- 小改动
- 明显提升直觉性
- 不破坏转化目标

的体验增强项。

## 实施建议

### 本轮建议落地内容

- 在 [src/pages/login/index.vue](../../../../shizi-frontend/src/pages/login/index.vue) 左上补一个弱化返回箭头。
- 复用现有 `goBack()` 逻辑。
- 保留当前底部“暂不登录”。
- 不额外修改登录成功后的流程。

### 本轮不做内容

- 不重做布局。
- 不调整登录主按钮层级。
- 不修改 onboarding 行为。
- 不做新的导航抽象组件。

## 最终结论

1. 登录页适合增加一个弱化的左上角返回箭头。
2. 该入口只保留图标，不加文案，不做完整顶栏。
3. 点击后沿用“优先返回上一页，兜底回首页”的现有逻辑。
4. 底部“暂不登录”应继续保留。
5. 这项优化属于体验增强，而不是缺陷修复。
