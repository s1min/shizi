# 趣字宝 UI 设计与样式风格规范 v3.1

> 视觉方向：Paper Classroom / 纸张学习册
> 适用范围：微信小程序、H5、uni-app + Vue 3 + Wot Design Uni
> 设计基线：`design/product_prototype_v3.1.md`
> 目标读者：前端工程师、交互设计师、内容设计师和验收人员

---

## 一、设计总原则

### 1.1 核心隐喻

趣字宝是一册会回应孩子的汉字学习册，不是游戏大厅，也不是家长数据后台。

- 纸张：代表稳定、专注、低刺激；
- 书签：代表今天从哪里开始；
- 田字格：代表汉字结构和书写练习；
- 贴纸：用于少量完成反馈，不用于无限收集；
- 铅笔线：用于边界、进度和内容分组；
- 声音按钮：让低龄儿童不依赖阅读文字完成任务。

### 1.2 双用户原则

产品必须同时服务家长和儿童，但不让两者争夺同一页面的注意力。

| 场景 | 默认用户 | 设计重点 |
|---|---|---|
| 登录、年龄、字库、每日计划 | 家长 | 信息清楚、风险可控、操作可撤销 |
| 今日任务启动 | 家长启动，孩子接管 | 一个主入口、语音说明、无复杂选择 |
| 字源、听音、描红、测验 | 儿童 | 大目标、大触控、少文字、即时反馈 |
| 结算页 | 儿童先看，家长后看 | 先给完成感，再给可解释结果 |
| 图鉴浏览 | 儿童和家长 | 轻浏览、状态清楚、不做无限游戏化 |
| 我的、报告、设置、隐私 | 家长 | 家长门槛、事实数据、明确下一步 |

### 1.3 年龄模式

所有儿童页面读取 `ageGroup`，至少支持三种模式：

| 模式 | 年龄 | 文字量 | 入口权限 | 题面表现 |
|---|---|---|---|---|
| `early` | 3-4 岁 | 极少 | 家长启动，儿童只学习 | 图片、语音、大图标 |
| `preschool` | 5-6 岁 | 少量 | 可从首页独立开始 | 汉字、图片、语音、少量拼音 |
| `school` | 7-8 岁 | 常规 | 可自主开始任务 | 汉字、拼音、组词、简单语境 |

年龄模式只改变呈现和默认流程，不改变学习结果、复习算法和隐私保护。

## 二、设计令牌

### 2.1 颜色令牌

颜色必须通过 SCSS/CSS 变量引用，不允许在页面组件中散落裸色值。

```scss
// Paper Classroom tokens
$paper: #FFF8F0;
$paper-deep: #F6EBDD;
$paper-grid: #EADBCB;
$surface: #FFFFFF;
$surface-soft: #FFFCF8;

$ink: #4A3728;
$ink-strong: #35271D;
$ink-muted: #876F5A;
$ink-light: #B19B86;

$apricot: #F5A623;
$apricot-dark: #B86E08;
$apricot-soft: #FFF0D4;
$sky: #5DADE2;
$sky-dark: #266C98;
$sky-soft: #EAF6FC;
$mint: #82C785;
$mint-dark: #397745;
$mint-soft: #ECF8EA;
$coral: #E97869;
$coral-dark: #A94339;
$coral-soft: #FFF0ED;

$line: #EADBCB;
$line-strong: #D8C3AE;
$disabled: #C9BDB2;
$overlay: rgba(53, 39, 29, 0.38);
```

语义映射：

| 语义 | 背景 | 前景/边框 | 适用 |
|---|---|---|---|
| 主操作 | `$apricot` | `$surface` 或 `$apricot-dark` | 开始学习、继续、确认 |
| 复习 | `$sky-soft` | `$sky-dark` | 待复习、复习入口、回顾 |
| 掌握 | `$mint-soft` | `$mint-dark` | 已掌握、完成、同步成功 |
| 待巩固 | `$coral-soft` | `$coral-dark` | 错题、待巩固、需要关注 |
| 中性 | `$surface-soft` | `$ink` / `$line` | 普通卡片、未开始 |
| 危险 | `$coral` | `$surface` | 删除、退出账号、不可恢复操作 |

状态不能只依靠颜色表达，必须同时使用文字、图标、形状或位置。

### 2.2 字体令牌

```scss
$font-display: 'Baloo 2', 'Noto Sans SC', sans-serif;
$font-body: 'Noto Sans SC', 'PingFang SC', 'Microsoft YaHei', sans-serif;
$font-hanzi: 'KaiTi', 'STKaiti', 'Noto Serif SC', serif;
$font-number: 'DIN Alternate', 'Noto Sans SC', sans-serif;
```

字体使用规则：

- `$font-hanzi` 只用于目标汉字、图鉴字格和田字格示范；
- `$font-display` 用于首页标题、完成标题和年龄卡标题；
- `$font-body` 用于按钮、说明、报告和错误提示；
- 不使用手写字体承载连续说明；
- 网络字体不可用时必须自然降级，不等待字体加载；
- 数字统计使用 `$font-number`，并启用等宽数字，避免数字变化导致布局跳动。

### 2.3 字号和行高

以下为 `rpx` 基准，H5 使用同等视觉比例：

| Token | rpx | 用途 | 行高 |
|---|---:|---|---:|
| `display-xl` | 104 | 目标汉字 | 1.0 |
| `display-lg` | 72 | 结果标题、页面主标题 | 1.15 |
| `display-md` | 52 | 卡片标题 | 1.25 |
| `title` | 36 | 章节标题、主按钮 | 1.35 |
| `body-lg` | 32 | 儿童语音提示辅助文字 | 1.5 |
| `body` | 28 | 普通正文、报告数据 | 1.55 |
| `label` | 24 | 标签、辅助说明 | 1.45 |
| `caption` | 22 | 次要说明、时间 | 1.4 |

正文不得低于 24rpx；儿童核心操作文字不得低于 30rpx。

### 2.4 间距、圆角和阴影

```scss
$space-1: 8rpx;
$space-2: 16rpx;
$space-3: 24rpx;
$space-4: 32rpx;
$space-5: 40rpx;
$space-6: 48rpx;
$space-7: 64rpx;
$space-8: 80rpx;

$radius-sm: 16rpx;
$radius-card: 28rpx;
$radius-sheet: 36rpx;
$radius-pill: 999rpx;

$shadow-card: 0 10rpx 28rpx rgba(112, 75, 34, 0.08), 0 2rpx 0 rgba(255, 255, 255, 0.8) inset;
$shadow-raised: 0 14rpx 32rpx rgba(112, 75, 34, 0.12), 0 2rpx 0 rgba(255, 255, 255, 0.72) inset;
$shadow-press: 0 4rpx 12rpx rgba(112, 75, 34, 0.08), 0 1rpx 0 rgba(112, 75, 34, 0.1) inset;
```

禁止：紫色渐变、纯黑厚重阴影、随机圆角、每个页面独立定义一套卡片效果。

### 2.5 安全区和触控

- 页面左右安全边距：32rpx；
- 底部 Tab 高度：112rpx，额外安全区不少于 24rpx；
- 所有可点击元素的视觉或扩展触控区不少于 88rpx × 88rpx；
- 相邻触控目标间距不少于 16rpx；
- 学习页底部主操作区固定高度 144-176rpx；
- 关闭图标可视尺寸可以是 40rpx，但按钮容器必须是 88rpx；
- 不使用仅靠左右滑动才能完成的关键动作；
- 需要滑动时必须同时提供可见按钮。

## 三、背景、插画和图标

### 3.1 纸张背景

页面使用两层背景：

1. `$paper` 作为基础颜色；
2. 极低透明度的点阵或纸纹作为装饰，透明度不超过 0.06。

推荐 CSS：

```scss
.paper-page {
  min-height: 100vh;
  background-color: $paper;
  background-image: radial-gradient(rgba(135, 111, 90, 0.08) 1rpx, transparent 1rpx);
  background-size: 24rpx 24rpx;
}
```

点阵不能用于学习内容区域的高对比背景，也不能干扰田字格和答题选项。

### 3.2 插画规则

- 插画采用扁平纸贴风格，轮廓使用 `$ink-muted`，不使用 3D 高光；
- 单页最多一个主插画和两个小装饰；
- 插画服务于任务理解，不单纯装饰；
- 低龄空状态优先使用具象物体，如书、铅笔、纸张；
- 不引入复杂角色 IP，避免后续版权和内容维护成本；
- 插画资源提供 `width`、`height` 或固定宽高比，防止加载跳动。

### 3.3 图标规则

- 统一使用同一套 SVG 线性图标，建议 Lucide/Carbon 风格；
- 线宽统一 2-2.5px，圆角端点；
- 不使用 Emoji 作为正式导航、状态和操作图标；
- 图标必须配合文字或语音，不依赖图标猜含义；
- 播放图标、锁图标、音量图标和勾选图标需要提供 disabled/pressed/success 三态。

## 五、页面 UI 规格

### 5.1 登录页 `pages/login/index.vue`

**页面目标**：让家长快速理解产品价值并安心授权。

**布局**：

```text
顶部安全区
品牌字标：趣字宝
副标题：每天认识几个字，慢慢记住它们
学习册插画：翻页 + 汉字贴纸
微信登录主按钮
本地体验次按钮
隐私摘要和协议链接
```

**视觉**：

- 背景 `$paper` + 右上角淡杏色圆形纸贴；
- 品牌字标使用 `$font-display` 52rpx；
- 插画最大宽度 520rpx，不使用全屏动画；
- 微信按钮使用品牌绿，仅用于微信授权，不纳入 Paper Classroom 主色；
- 体验按钮为 `$surface-soft` 描边按钮。

**交互**：

- 授权按钮 loading 时保持尺寸不变；
- 协议未阅读不默认勾选；
- 登录失败在按钮下方显示原因和重试，不使用 toast 作为唯一反馈；
- 体验模式明确显示“仅保存在本机”。

### 5.2 首次引导 `pages/onboarding/index.vue`

**三步家长向导**：年龄 → 字库 → 每日计划。

顶部进度使用 3 个书签节点，当前节点为 `$apricot`，已完成节点为 `$mint`，并配文字标签。

年龄卡：

- 3-4 岁：纸飞机插画，副文案“图片和声音为主”；
- 5-6 岁：铅笔插画，副文案“认读和描红一起练”；
- 7-8 岁：书本插画，副文案“同步单元，加入简单语境”。

选择后显示推荐字库，推荐卡使用书签色条和“推荐”标签。切换字库前显示学习进度保留说明。

完成页文案：`设置完成，把手机交给孩子吧`，主按钮：`进入今天的学习`。

### 5.3 首页 `pages/home/index.vue`

**核心原则**：家长确认任务，孩子点击一个入口开始。

顶部：

- 左侧：`小明的学习册`；
- 右侧：家长锁图标；
- 第二行：`今天是第 6 天 · 已掌握 18 字`；
- 低龄模式可将统计文案折叠为“今天学习 5 分钟”。

主卡：

- 卡片背景 `$surface`，顶部有 16rpx 高 `$apricot` 书签条；
- 标题：`今天学什么？`；
- 任务摘要：`先复习 2 个字，再学 3 个新字`；
- 主 CTA：`开始今天的学习`；
- 右上角播放按钮，播放一句任务说明。

状态：

- 首次：`打开今天的第一课`；
- 中断：`继续学习` + `上次到：听音辨认`；
- 已完成：薄荷绿卡片 + `今天完成啦`；
- 离线：顶部纸签提示“离线学习，联网后同步”。

儿童模式下不展示并列的“开始复习/开始新字”按钮，系统在主 CTA 后自动编排；7-8 岁可以点击摘要展开详情，但仍保留一个主 CTA。

### 5.4 学习壳层 `subpkg-learning/learn/index.vue`

隐藏 TabBar，统一使用 `LearnFlowHeader`：

```text
左：关闭（88rpx 触控区）
中：步骤名 + 当前字序号
右：1/3 或步骤状态
下：四步节点：看一看 / 听一听 / 写一写 / 小测验
```

学习内容区垂直居中，底部固定操作区不随内容跳动。

### 5.5 字源页 `CharCard.vue`

- 动画内容区 520-600rpx 高；
- 目标字使用 `display-xl`，颜色 `$ink-strong`；
- 解释文案最多 1 行，低龄模式使用音频优先；
- `再看一遍`为描边次按钮；
- `我看懂了`为唯一主按钮；
- 动画结束后按钮从下方 16rpx + opacity 0 渐入；
- 不自动跳转，不因重复播放增加学习次数。

### 5.6 听音页 `SpeakPractice.vue`

- 播放按钮直径不少于 112rpx，背景 `$sky-soft`，图标 `$sky-dark`；
- 题干支持语音播报，文字仅作辅助；
- 选项卡最小高度 128rpx，三列或 2+1 布局按设备宽度决定；
- 选项正常态 `$surface` + `$line`；
- 选中态 `$apricot-soft` + `$apricot-dark`；
- 正确态 `$mint-soft` + 勾选图标 + “答对了”；
- 错误态 `$coral-soft` + “再听一次”，不出现刺眼全屏红色；
- 选项点击后锁定 500ms 以内，反馈完成后可重试。

年龄差异：

- `early`：图片和声音，汉字可作为小标签；
- `preschool`：图片 + 大汉字；
- `school`：大汉字 + 拼音干扰项。

### 5.7 描红页 `TracingPractice.vue`

- 田字格卡片背景 `$surface-soft`；
- 格线使用 `$paper-grid`，线宽不超过 2rpx；
- 笔顺演示使用 `$sky`，可写轨迹使用 `$apricot`；
- `清除`为中性按钮，`看笔顺`为复习蓝按钮，`写好了`为主按钮；
- 3-4 岁显示“可以跳过”，5-8 岁显示“开始描红”；
- 触控容错优先于笔画精确度；
- 无法加载 HanziWriter 时自动切换笔顺动画模式。

### 5.8 测验页 `QuizCard.vue` / `unit-test.vue`

- 单题单屏；
- 顶部显示 `还剩 X 题` 和进度条，不依赖星星做唯一反馈；
- 选项区域至少占内容区 40%；
- 每题结束显示局部反馈，避免遮挡整个题面；
- 测试结束统一进入结果页；
- 错题列表使用 `$coral-soft` 纸签，标注“需要再看一看”。

### 5.9 复习页 `subpkg-learning/review/index.vue`

视觉主题切换到 `$sky`，但仍使用同一套纸张卡片和圆角。

- 顶部：返回、`今日复习`、`2/5`；
- 中央闪卡最大宽度 620rpx，最小高度 680rpx；
- 正面：大字或图片，不显示拼音；
- 翻面：拼音、组词、播放按钮；
- 底部：`不认识`（珊瑚描边）和 `认识`（薄荷实底）；
- 两按钮均不少于 240rpx 宽，间距 24rpx；
- 左右滑动只作为辅助，不隐藏底部按钮；
- 复习完成时显示“今天复习完成”，不自动追加下一组。

### 5.10 结算页 `unit-complete.vue`

- 页面中心出现一张“今日学习记录纸”；
- 使用薄荷绿勾选和少量贴纸，不使用烟花、撒花和大面积粒子；
- 展示：接触字数、即时正确率、待巩固字、学习时长、下次复习；
- 3-6 岁先显示“今天完成啦”，7-8 岁可展开数据；
- 主 CTA：`结束学习`；
- 次 CTA：`复习待巩固字`；
- 家长 CTA：`查看今日报告`，必须经过家长状态或从结果页单独进入。

### 5.11 图鉴页 `pages/loot/index.vue`

- 顶部显示 `已掌握 18 / 已接触 24`；
- 进度条使用 `$mint`，待巩固数量使用 `$coral` 文字和图标；
- 筛选条：`全部 / 已掌握 / 待巩固`；
- 网格 4 列，字格比例 1:1；
- 已掌握：`$mint-soft` 纸贴；
- 初步掌握：`$apricot-soft`；
- 待巩固：`$coral-soft` + 小提示标记；
- 未开始：灰纸色 + 问号，不展示空白透明卡；
- 详情使用底部 sheet，含读音、组词、字源重播和下次复习时间。

### 5.12 家长中心 `pages/me/index.vue`

进入 `我的` 时先做家长验证，再展示：

- 宝宝档案和年龄；
- 当前字库；
- 今日学习时长；
- 待巩固字；
- 今日报告入口；
- 学习单元管理；
- 学习设置、提醒、隐私和数据。

家长中心不使用儿童端的大字游戏化标题，采用 `$font-body` + 纸白卡片 + 左对齐信息层级。

### 5.13 家长报告页

页面结构：

```text
今日概览
├── 学习时长
├── 新学字
├── 完成复习
└── 待巩固字

记忆情况
├── 即时答对率
└── 延迟回忆率

下一步
├── 待巩固字列表
└── 开始巩固
```

- 用两列数据卡 + 一条水平进度条即可，不做复杂仪表盘；
- 每个指标有标签和解释，例如“延迟回忆率：第二天还能认出的字”；
- 图表颜色不使用红绿唯一组合，配合图标和文字；
- 空数据使用“还没有复习记录，完成一次复习后这里会显示结果”；
- 唯一主 CTA 为“开始巩固”。

### 5.14 协议和隐私页

- 使用家长工具栏和正文排版，不使用儿童插画装饰；
- 正文 28rpx，行高 1.75；
- 章节标题 `$ink-strong` 36rpx；
- 数据删除使用独立危险区，必须二次确认；
- 隐私摘要与完整政策分层展示，首次授权只展示摘要并提供全文入口。

## 六、交互和动效规范

### 6.1 动效令牌

```scss
$motion-fast: 150ms;
$motion-base: 220ms;
$motion-slow: 360ms;
$ease-standard: cubic-bezier(0.22, 0.8, 0.24, 1);
$ease-exit: cubic-bezier(0.4, 0, 1, 1);
```

### 6.2 必做动效

- 卡片按下：`scale(0.98)`，150ms；
- 主按钮按下：内阴影加深，恢复 180ms；
- 页面进入：主卡片 `opacity + translateY(16rpx)`，220ms；
- 字源动画：由内容脚本定义，不能超过 10 秒且不阻止操作；
- 复习翻卡：Y 轴旋转或交叉淡入，240ms；
- 列表首次出现：每项间隔 40ms，最多 5 项；
- 结果状态：勾选图标缩放一次，不连续跳动。

### 6.3 禁止动效

- 无限循环的背景动画；
- 自动播放多个连续页面动画；
- 用撒花、烟花或粒子阻塞主操作；
- 答错时全屏震动、红屏或强烈音效；
- 依赖 hover 的交互；
- 同一页面超过两个主要动效焦点。

### 6.4 无障碍和低刺激

- 遵守 `prefers-reduced-motion`，在 H5 关闭非必要动效；
- 声音关闭时所有任务必须有文字或图像替代；
- 正文和按钮文字对比度达到 4.5:1；
- 色彩状态同时配合文字和图标；
- 反馈在 100ms 内出现可见变化；
- 不用过小的图标承载唯一任务入口。

## 七、状态组件规范

### 7.1 Button

| 类型 | 背景 | 文本 | 使用 |
|---|---|---|---|
| Primary | `$apricot` | `$surface` | 每页唯一主 CTA |
| Secondary | `$surface` | `$apricot-dark` | 次要动作 |
| Review | `$sky-soft` | `$sky-dark` | 复习、再听一次 |
| Success | `$mint` | `$surface` | 已完成、保存成功 |
| Danger | `$coral` | `$surface` | 删除、危险操作 |
| Ghost | transparent | `$ink-muted` | 返回、跳过、稍后 |

按钮规则：

- 高度 88rpx；
- 主按钮宽度默认填充容器，次按钮不小于 220rpx；
- 只有一个 Primary；
- loading 时显示小型进度并禁用重复点击；
- disabled 透明度 0.45，仍保留文字可读性。

### 7.2 Card

- 默认 `$surface` + `$shadow-card` + `$radius-card`；
- 互动卡片按下后使用 `$shadow-press`；
- 状态卡通过顶部书签色条表达主题；
- 不在同一页面混用直角、胶囊和超大圆角卡片。

### 7.3 Empty / Error / Offline

每个状态组件必须包含：

1. 状态图标或小插画；
2. 原因说明；
3. 一个可执行动作；
4. 必要时提供“稍后再说”。

示例：

- 空复习：`今天没有到期复习` + `先学新字`；
- 同步失败：`暂时没有同步成功` + `重试同步`；
- 音频失败：`声音没有准备好` + `看文字继续`；
- 中断恢复：`上次学习到：听音辨认` + `继续学习`。

## 八、前端实现约束

### 8.1 SCSS 组织

建议新增：

```text
src/style/
├── tokens.scss       // 颜色、字号、间距、圆角、阴影、动效
├── paper-theme.scss   // 页面背景、卡片、书签、田字格
├── components.scss    // button/card/status/tabbar 公共类
└── index.scss         // 统一入口
```

页面组件只使用语义类和变量，例如：

```scss
.task-card {
  background: $surface;
  border-radius: $radius-card;
  box-shadow: $shadow-card;
}

.task-card__bookmark {
  background: $apricot;
}

.task-card__review {
  background: $sky-soft;
  color: $sky-dark;
}
```

### 8.2 组件命名建议

```text
PaperPage
PaperCard
PaperBookmark
PaperButton
PaperStatusBadge
PaperProgress
ParentGate
LearningShell
LearnFlowHeader
AudioPrompt
CharacterTile
ReviewFlashCard
ReportMetricCard
```

现有组件可渐进迁移，不要求一次性重命名；新组件不得再引入紫色主题或独立阴影体系。

### 8.3 数据和视觉分离

页面通过以下状态决定视觉，不在模板中直接猜测：

```ts
type AgeGroup = 'early' | 'preschool' | 'school'
type CharacterStatus = 'unseen' | 'exposed' | 'emerging' | 'needs_review' | 'mastered' | 'stable'
type TaskState = 'first' | 'resume' | 'review' | 'new' | 'complete' | 'offline'
```

推荐提供：

```ts
const uiMode = computed(() => getUiMode(ageGroup.value))
const taskState = computed(() => getTodayTaskState(learnStore))
const charStatus = computed(() => getCharacterStatus(record))
```

视觉组件只接收 `status`、`variant` 和 `ageGroup`，不直接读取 store 内部复杂结构。

### 8.4 性能

- 首屏只加载 Logo、首页主卡和必要图标；
- 字源动画、图鉴详情和复杂报告按页面/交互懒加载；
- 图片使用 WebP，声明尺寸；
- 不在每个卡片中重复加载相同 SVG 或大图；
- 报告图表在无数据时使用空状态，不渲染空坐标轴；
- 纸纹使用 CSS 点阵优先，避免加载大纹理图片；
- 学习流程的内容区域预留固定高度，避免音频/图片加载造成跳动。

## 十、交付和迁移顺序

建议前端按以下顺序落地，避免一次性改动所有页面：

1. 建立 `tokens.scss`，统一颜色、字体、间距、圆角和阴影；
2. 重构 `Tabbar`，将底部入口统一为今日/图鉴/我的；
3. 重构首页主任务卡和低龄模式；
4. 统一 `LearnFlowHeader`、底部操作区和学习页背景；
5. 重构字源、听音、描红和测验的状态组件；
6. 重构复习闪卡为天空蓝主题；
7. 重构结算页和图鉴状态；
8. 为 `me` 增加家长门槛分流和报告入口；
9. 最后处理协议页、空状态、离线和错误状态；
10. 按页面验收清单进行 375/390/414px 真机回归。

## 十一、最终设计结论

趣字宝的统一视觉风格不是“所有页面都变得可爱”，而是让不同用户在同一套纸张学习册语言中获得不同的信息密度：

- 家长看到清楚、可信、可操作的学习任务和报告；
- 3-6 岁儿童看到大图、声音和唯一主入口；
- 7-8 岁儿童逐步看到文字、进度和自主复习；
- 所有页面共享暖杏、天空蓝、薄荷绿、纸张背景和统一动效节奏；
- 所有重要状态都有文字和图标解释，不用颜色或游戏刺激替代教育反馈。

最终交互原则：

> 家长负责选什么、学多少、是否继续；孩子负责看、听、点、写、回忆。
