# 产品原型设计方案 v2：轻量级趣味识字伴侣

> 基于 v1 方案的专业评审与改进，从儿童教育产品经理和UI交互设计师双重视角优化。

---

## 一、v1 方案评审总结

### 1.1 优势保留

- 产品定位精准："轻量级趣味识字伴侣"切中洪恩（太重游戏）和叫叫（太像上课）之间的市场空白
- 字源动画按汉字类型分层（象形/会意/抽象）是专业的教学设计
- 多题型测试池（A/B/C/D）比单一选择题更科学
- 数据库"元数据与编排分离"架构清晰
- 素材来源分阶段策略务实

### 1.2 核心问题诊断

| # | 问题 | 影响 | 改进方向 |
|---|------|------|----------|
| 1 | **首次体验缺失** | 新用户无引导，家长不知从何开始，流失率高 | 新增"30秒体验流程" |
| 2 | **学习流程过长且无弹性** | 6步线性流程对3岁太长、对6岁太简单 | 引入自适应节奏机制 |
| 3 | **复习入口太深** | 藏宝阁作为独立Tab，用户容易只学新字不复习 | 首页强化复习提醒 |
| 4 | **激励体系碎片化** | 星星、徽章、等级、智慧豆多套货币，3-6岁理解不了 | 统一为"星星"单一货币 |
| 5 | **亲子互动场景薄弱** | 家长仅限"设置+看报告"，缺少参与感 | 新增亲子共学模式 |
| 6 | **底部导航缺少高频入口** | 3个Tab缺少"今日任务"直达 | 改为4Tab+首页任务卡 |
| 7 | **MVP技术栈偏重** | MySQL+MongoDB双数据库维护成本高 | MVP阶段统一用PostgreSQL |

## 二、产品概述（修订版）

*   **产品名称**：趣字宝 / QuZiBao
*   **平台**：微信小程序 (MVP)，后续扩展 App
*   **核心理念**：极简高效、字源解说、亲子驱动
*   **目标用户**：3-6岁学龄前儿童及其家长
*   **核心差异化**：
    1.  克制的游戏化 — 保留字源动画和即时反馈，砍掉复杂游戏
    2.  亲子共学 — 家长不只是旁观者，而是学习伙伴
    3.  自适应节奏 — 根据孩子年龄和表现动态调整学习步骤

## 四、详细页面原型设计

### 4.1 新用户引导流程（新增）

**设计目标**：30秒内让家长理解产品价值，60秒内让孩子开始第一次学习。

**流程**：

1.  **微信授权登录**（1步完成，不要求手机号）
2.  **选择宝宝年龄段**：
    *   3-4岁 → 自动推荐"学前启蒙"字库，每日3字
    *   5岁 → 推荐"学前启蒙"，每日5字
    *   6岁+ → 推荐"一年级上册"，每日5字
3.  **体验第一个字**：直接进入"日"字的字源动画（不需要任何额外操作）
4.  **完成体验 → 引导创建宝宝档案**

**交互细节**：
- 年龄选择用3个大卡片，每个卡片配一个可爱的动物形象
- 不设"跳过"按钮，但允许左滑快速通过
- 体验字固定为"日"（最简单的象形字，动画效果最直观）

---

### 4.2 首页：今日任务（重新设计）

**设计目标**：一屏展示"今天该做什么"，零思考成本。

**布局**：

*   **顶部状态栏**：
    *   左侧：宝宝头像 + 昵称 + 等级称号
    *   右侧：连续打卡天数（火焰图标🔥 + 数字）

*   **核心区域：任务卡片（纵向排列）**：

    **卡片 1：复习任务（优先展示）**
    *   条件：有待复习的字时显示
    *   样式：柔和的蓝色卡片
    *   内容："今天有 X 个字需要复习"
    *   按钮："开始复习"（进入闪卡模式）
    *   **改进说明**：复习任务置顶，解决原方案"复习入口太深"的问题

    **卡片 2：新字学习**
    *   样式：温暖的黄色卡片
    *   内容：当前字库名 + 当前单元名 + "还有 X 个新字"
    *   按钮："开始学习"
    *   进度条：显示当前单元完成百分比

    **卡片 3：魔法造句（条件展示）**
    *   条件：累计学满 10 个字后解锁
    *   样式：紫色渐变卡片，带"魔法书"图标
    *   内容："用学过的字造个句子吧"

*   **底部浮动区域**：
    *   字库切换入口（小字显示当前字库名，点击切换）

**与原方案对比**：
- 原方案首页是"地图式"，需要孩子理解"岛屿→阶段→单元"的层级关系，认知负担重
- 新方案首页是"任务卡片式"，直接告诉用户"今天做什么"，零思考成本
- 地图式设计保留在"图鉴"Tab中，作为成就展示而非主导航


---

### 4.3 核心学习流程（优化版）

**设计目标**：单字闭环 2-4 分钟，支持自适应跳步。

**自适应机制（新增）**：
- 系统根据宝宝年龄和历史表现，动态决定每个字需要经过哪些步骤
- 3-4岁：完整4步（动画→认读→描红→测验）
- 5-6岁且该字类型为"抽象字"：可跳过描红（抽象字描红意义不大）
- 复习模式：仅保留"认读+测验"两步

#### 步骤 1：字源动画 (Origin Animation) — 保留原方案

原方案的分层设计（象形/会意/抽象）非常专业，完整保留。

**微调**：
- 动画时长严格控制在 8-12 秒，不超过 15 秒
- 动画结束后自动跳转下一步（不需要点"下一步"按钮），减少操作步骤
- 新增"再看一遍"小按钮（左下角），满足部分孩子重复观看的需求

#### 步骤 2：认读互动 (Listen & Speak) — 优化

**界面**：
*   屏幕中央：大字展示（占屏幕 40% 高度）
*   字的上方：拼音标注（带声调）
*   字的下方：一个组词（如"火车"），帮助建立语义联结

**交互优化**：
*   系统自动播放读音（不需要用户点击播放）
*   **简化跟读**：点击麦克风按钮开始录音，松手结束（原方案"长按"对3岁孩子太难）
*   **降级方案**：如果用户拒绝麦克风权限，改为"点击正确读音"的选择题（播放3个不同读音，选对的）
*   反馈保持原方案：成功撒花，失败温柔提示

#### 步骤 3：趣味描红 (Tracing) — 优化

**保留原方案核心设计**，微调：
*   **笔顺引导更强**：第一笔自动演示完整笔画路径（半透明绿色引导线），后续笔画只显示起点
*   **容错度提高**：3-4岁模式下，笔迹偏离容忍度放宽至田字格宽度的 30%（原方案过于严格会打击积极性）
*   **快速通过**：如果孩子已经在之前的字库中写过这个字（跨字库重复字），显示"你已经会写了，跳过？"

#### 步骤 4：即时测验 (Mini Quiz) — 优化

**保留原方案的多题型池设计**（A/B/C/D 四种题型），这是原方案的亮点。

**优化点**：
*   每个字固定出 1 道题（原方案 1-2 道，减少到 1 道降低疲劳感）
*   **题型选择逻辑优化**：
    *   象形字/具象名词 → 优先题型 B（看图选字），因为字形辨识是核心能力
    *   抽象字 → 优先题型 C（听音选字）
    *   形近字同时出现在本单元 → 必出题型 B 对比题
    *   已学 50 字以上 → 解锁题型 D（语境选字）
*   **错误处理优化**：答错后不仅高亮正确答案，还自动重播该字的读音，强化"音-形"联结

#### 步骤 5：单元闯关 (Unit Quiz) — 保留原方案

原方案设计完善，保留。

**微调**：
*   题目数量 = 单元字数（不多不少），每字一题
*   增加"连对加速"机制：连续答对 3 题，出题速度加快，营造心流感
*   结果评级简化为两档：
    *   全对 → 3 星 + "单元之星"徽章
    *   未全对 → 1-2 星（按正确率），错题自动加入复习队列

#### 步骤 6：结算打卡 (Result & Share) — 优化

**保留打卡海报功能**，优化：
*   海报自动生成，但不强制弹出分享（避免打断孩子的成就感体验）
*   先展示"恭喜"动画（2秒），再展示成果数据，最后底部出现"分享给家人"按钮
*   **新增"亲子时刻"提示**：如果当天是第一次学习，结算页底部显示"和爸爸妈妈一起读今天学的字吧"，引导亲子互动

### 4.4 复习系统（重新设计）

**设计目标**：让复习像"刷卡片"一样轻松，不给孩子压力。

#### 4.4.1 闪卡复习模式

**触发**：首页"复习任务"卡片点击进入。

**界面**：
*   全屏卡片，左右滑动切换
*   卡片正面：大字展示（无拼音）
*   点击卡片 → 翻转 → 背面显示：拼音 + 组词 + 小插图
*   底部两个按钮：
    *   "认识" ✓（绿色）→ 复习等级 +1
    *   "不认识" ✗（灰色）→ 复习等级归零，加入明日复习

**与原方案对比**：
- 原方案复习在"藏宝阁"Tab 中，需要用户主动进入
- 新方案复习在首页任务卡中优先展示，且交互极简（左右滑 + 两个按钮）

#### 4.4.2 艾宾浩斯复习间隔（保留原方案）

| 复习等级 | 间隔 |
|:---:|:---:|
| 0 | 当天再次出现 |
| 1 | 1 天后 |
| 2 | 2 天后 |
| 3 | 4 天后 |
| 4 | 7 天后 |
| 5 | 15 天后 |
| 6 | 30 天后 |
| 7 | 已掌握（不再出现） |

**优化**：原方案 level 0 间隔为"1小时后"，改为"当天再次出现"。因为目标用户每天只学习一次（15-30分钟），1小时后大概率已经退出小程序。

---

### 4.5 汉字图鉴（原"藏宝阁"重新设计）

**设计目标**：把"已学汉字"变成可收藏、可展示的成就。

**布局**：

*   **顶部统计**：
    *   "已收集 128 / 300 个汉字"（进度条）
    *   当前字库名称

*   **汉字墙（网格展示）**：
    *   每个格子是一个汉字卡片
    *   **已掌握**：金色卡片，点击可查看详情（拼音、组词、字源动画回放）
    *   **学习中**：白色卡片，显示复习进度环
    *   **未学习**：灰色剪影，显示"?"

*   **魔法造句入口**：
    *   页面底部固定按钮："用这些字造句"
    *   点击进入 AI 造句功能（保留原方案设计）

---

### 4.6 星星商店（新增）

**设计目标**：统一激励体系，给星星一个"花出去"的地方。

**改进说明**：
- 原方案有星星、徽章、等级称号、智慧豆多套激励，3-6岁孩子理解不了
- 新方案统一为"星星"单一货币：学习得星星 → 星星兑换奖励

**可兑换内容**：
| 奖励 | 星星数 | 说明 |
|------|--------|------|
| 新头像框 | 10 星 | 12 生肖系列 |
| 称号升级 | 30 星 | 识字小童生 → 识字秀才 → 识字举人 → 翰林学士 |
| 主题皮肤 | 50 星 | 更换学习界面背景色（森林绿/海洋蓝/星空紫） |
| 解锁彩蛋动画 | 20 星 | 特殊的字源动画（如"龙"字的动画） |

**交互**：
- 商店界面用"扭蛋机"或"宝箱"的视觉隐喻，增加趣味性
- 兑换时有开箱动画，满足孩子的期待感

---

### 4.7 我的（优化版）

#### 4.7.1 宝宝档案
保留原方案设计，微调：
- 头像选择从"12生肖"扩展为"12生肖 + 可用星星解锁的头像框"
- 等级称号改为星星商店兑换，不再自动升级

#### 4.7.2 学习数据（原"学习日报"优化）

**简化数据展示**：
*   **今日**：新学 X 字 | 复习 X 字 | 学习 X 分钟
*   **本周**：简单柱状图（7天学习字数）
*   **总计**：累计识字 X 个

**去掉"超过 X% 小朋友"的排名**：
- 理由：儿童教育产品不应引入竞争焦虑，这与"轻松学习"的产品理念矛盾
- 替代：改为"你已经认识了一本绘本里的大部分字了"等正向鼓励文案

#### 4.7.3 家长中心
保留原方案设计，补充：
*   **安全锁**：改为"滑动解锁"（向右滑动一个滑块），比算术题更快且同样防误触
*   **新增"亲子模式"开关**：开启后，每次学习结束会弹出"亲子互动"提示卡

## 五、视觉设计风格指南（修订版）

### 5.1 色彩体系优化

原方案的暖黄 (#FFD700) 饱和度过高，长时间注视容易视觉疲劳。调整为低饱和度暖色系：

| 用途 | 颜色 | 色值 | 说明 |
|------|------|------|------|
| 主色 | 暖杏色 | #F5A623 | 主按钮、重要元素，比纯金色更柔和 |
| 辅助色 | 天空蓝 | #5DADE2 | 复习相关元素、辅助按钮 |
| 背景色 | 奶油白 | #FFF8F0 | 页面背景，比纯白更温暖 |
| 成功色 | 薄荷绿 | #82C785 | 答对反馈、已掌握状态 |
| 提示色 | 薰衣草紫 | #BB8FCE | 魔法造句、特殊功能 |
| 文字色 | 深棕 | #4A3728 | 正文文字，比纯黑更柔和 |

**护眼原则**：
- 所有颜色饱和度不超过 70%
- 背景避免纯白 (#FFFFFF)，统一使用暖色调底色
- 动画避免高频闪烁（闪烁频率 < 3Hz）

### 5.2 字体规范

| 场景 | 字体 | 字号 | 说明 |
|------|------|------|------|
| 汉字主体（学习页） | 楷体 (KaiTi) | 120rpx+ | 笔画清晰，适合儿童辨识 |
| 拼音标注 | 系统默认 | 36rpx | 标注在汉字上方 |
| UI 文字 | 系统圆体 | 28-32rpx | 按钮、标签等 |
| 家长端文字 | 系统默认 | 28rpx | 设置、数据等 |

### 5.3 插画与动效

保留原方案的"扁平国潮风"和分阶段素材策略，补充动效规范：

*   **页面转场**：统一使用 300ms ease-out 滑动动画
*   **按钮反馈**：点击时 scale(0.95) + 100ms 回弹
*   **成功动效**：星星从按钮位置飞向顶部星星计数器（路径动画）
*   **错误动效**：卡片左右摇晃 2 次（shake 动画），不使用红色闪烁（避免惊吓）

## 六、交互设计原则（补充版）

保留原方案的3条原则（大触控区、即时反馈、防误触），补充：

4.  **单手可操作**：所有核心交互（点击、滑动）都在屏幕下半部分 60% 区域内，适配孩子握持手机/平板的姿势
5.  **无文字依赖**：所有按钮同时使用图标+文字，确保不识字的孩子也能通过图标理解功能
6.  **退出保护**：学习流程中按返回键，弹出确认弹窗"要休息一下吗？"，而非直接退出（防止误触丢失进度）
7.  **音频优先**：所有指令和反馈都有语音版本，不依赖文字阅读。系统语音使用温柔的童声（非机械TTS）
8.  **触控热区**：按钮最小触控区域 88rpx × 88rpx（原方案 80rpx 偏小），间距不小于 24rpx

## 七、数据库设计（修订版）

### 7.1 技术选型调整

**MVP 阶段改为单数据库方案**：统一使用 PostgreSQL，用 JSONB 字段存储灵活结构的汉字数据。

**调整理由**：
- 原方案 MySQL + MongoDB 双数据库对个人开发者维护成本过高（两套连接、两套备份、两套监控）
- PostgreSQL 的 JSONB 类型完全能胜任汉字元数据的灵活存储需求，同时保留关系型数据库的事务和约束能力
- 后续如果数据量增长到需要分离，再迁移不迟（MVP 阶段总数据量极小）

**修订后技术栈**：

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 运行时 | Node.js (v20 LTS) | 升级到最新 LTS |
| Web 框架 | NestJS | 比 Koa2 更结构化，内置依赖注入、模块化，适合后续扩展 |
| ORM | Prisma | 类型安全、自动迁移 |
| 数据库 | PostgreSQL 16 | 单库方案，JSONB 存储灵活数据 |
| 鉴权 | JWT + 微信登录 | 不变 |
| 部署 | Docker Compose | PostgreSQL + Node.js |
| 服务器 | 轻量云 2C2G | 约 50-100 元/月 |

### 7.3 查询方式

与原方案逻辑一致，但简化为单库查询：

```javascript
// 获取单元学习数据（单库查询，无需跨库）
async function getUnitChars(libId, unitId, childId) {
  // 1. 获取字库编排
  const library = await prisma.library.findUnique({ where: { id: libId } })
  const unit = library.stages
    .flatMap(s => s.units)
    .find(u => u.id === unitId)
  const charList = unit.chars

  // 2. 批量获取汉字元数据
  const characters = await prisma.character.findMany({
    where: { char_id: { in: charList } }
  })

  // 3. 获取该孩子的进度
  const progress = await prisma.userProgress.findMany({
    where: {
      child_id: childId,
      library_id: libId,
      char_id: { in: charList }
    }
  })

  return { unit, characters, progress }
}
```

// GET /api/v1/progress/:childId/today
// 返回今日任务概览，供首页渲染
async function getTodayTasks(childId) {
  const child = await prisma.child.findUnique({ where: { id: childId } })
  
  // 待复习字数
  const reviewCount = await prisma.userProgress.count({
    where: {
      child_id: childId,
      next_review_at: { lte: new Date() },
      status: { in: [1, 3] }  // 学习中或需复习
    }
  })
  
  // 当前单元进度
  const library = await prisma.library.findUnique({ where: { id: child.current_library } })
  const currentUnit = library.stages
    .flatMap(s => s.units)
    .find(u => u.id === child.current_unit)
  
  const learnedInUnit = await prisma.userProgress.count({
    where: {
      child_id: childId,
      library_id: child.current_library,
      char_id: { in: currentUnit?.chars || [] },
      status: { gte: 1 }
    }
  })
  
  return {
    review_count: reviewCount,
    current_library: child.current_library,
    current_unit: currentUnit,
    unit_progress: {
      learned: learnedInUnit,
      total: currentUnit?.chars?.length || 0
    },
    streak_days: child.streak_days
  }
}

## 十、改进总结

| 维度 | 原方案 | 改进方案 | 改进理由 |
|------|--------|----------|----------|
| 首次体验 | 无 | 30秒引导 + 体验第一个字 | 降低流失率 |
| 首页设计 | 地图式（岛屿/书架） | 任务卡片式 | 零思考成本，直接告诉用户做什么 |
| 复习入口 | 独立Tab（藏宝阁） | 首页优先展示 | 解决"只学不复习"问题 |
| 学习节奏 | 固定6步 | 自适应跳步 | 适配不同年龄和字类型 |
| 激励体系 | 星星+徽章+等级+智慧豆 | 统一为星星+星星商店 | 3-6岁理解单一货币 |
| 底部导航 | 3 Tab | 4 Tab | 增加图鉴和星星商店入口 |
| 数据库 | MySQL + MongoDB | PostgreSQL (JSONB) | MVP阶段降低运维成本 |
| Web框架 | Koa2 | NestJS | 更结构化，适合后续扩展 |
| 色彩 | 高饱和暖黄 #FFD700 | 低饱和暖杏 #F5A623 | 护眼，降低视觉疲劳 |
| 排名机制 | "超过X%小朋友" | 正向鼓励文案 | 儿童教育不应引入竞争焦虑 |
