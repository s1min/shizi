# 趣字宝产品原型设计方案 v3.1：学习效果与家长治理版

> 本文档基于 `design/product_prototype_v3.md` 修订，作为产品评审和下一轮研发实施基线。
> v3.1 不推翻个人开发者 MVP 的范围，而是补齐儿童教育产品必须具备的五个约束：适龄、掌握、复习、家长可控、效果可测。
>
> 目标不是增加功能数量，而是确保产品能够回答三个问题：
>
> 1. 孩子是否真的记住了这个字？
> 2. 家长是否知道孩子哪里需要帮助？
> 3. 产品是否能在不诱导沉迷的情况下形成稳定学习习惯？

---

## 一、版本定位

### 1.1 继承 v3 的决策

以下决策继续有效：

- 平台优先微信小程序，保留 H5 体验模式；
- 技术栈继续使用 uni-app + Vue 3、Pinia、NestJS、MongoDB；
- 以字源动画作为核心差异化，不与重 3D 游戏产品竞争；
- MVP 保留单字学习、单元测试、进度保存和复习；
- 星星商店、AI 造句、ASR 评分、复杂亲子社交和自适应学习继续延后；
- 采用低饱和暖色视觉，控制动效和使用时长。

### 1.2 v3.1 的核心修订

| 领域 | v3 现状 | v3.1 决策 |
|---|---|---|
| 用户分层 | 3-6 岁共用固定流程 | 3-4、5-6、6 岁以上三档基础分层 |
| 学习结果 | 完成流程即计入已学 | 区分已接触、初步掌握、待巩固、已掌握、稳定掌握 |
| 复习 | 文档与实现间隔不一致 | 固定间隔、答错降级、延迟回忆规则统一 |
| 家长能力 | 安全锁和报告偏后置 | 家长入口、基础报告、时长边界列入 MVP |
| 每日任务 | 只有新学和复习卡片 | 明确“先复习、后新学”的任务调度规则 |
| 内容质量 | 有字源和例句字段 | 增加来源、年龄、审核、发音和版本字段 |
| 指标 | 主要看字数、星星、打卡 | 增加首字完成、延迟回忆、复习完成和留存指标 |
| 文档口径 | 原型、代码、验收报告有偏差 | 建立唯一产品基线和差异清单 |

### 1.3 产品定位

**产品名称**：趣字宝 / QuZiBao

**定位**：面向 3-6 岁儿童的轻量汉字启蒙工具，通过字源、声音、图像和短时复习，帮助儿童建立基础汉字识别；通过家长报告，让监护人知道孩子哪些字已经记住、哪些字仍需巩固。

**不做的承诺**：

- 不承诺通过一次学习就“学会”汉字；
- 不承诺替代幼儿园、家庭教育或小学语文课程；
- 不以连续使用时长、游戏等级或虚拟奖励作为主要学习成果；
- 不在 MVP 阶段提供未经审核的 AI 生成儿童内容。

## 二、用户与使用场景

### 2.1 用户角色

| 角色 | 主要任务 | 关键诉求 |
|---|---|---|
| 儿童 | 完成当天少量学习和复习 | 看得懂、点得准、反馈及时、不被文字说明卡住 |
| 家长/监护人 | 选择内容、控制时长、查看效果 | 适龄、内容准确、过程可控、结果可信 |
| 产品运营/内容编辑 | 编排字库和教学内容 | 可审核、可版本化、可回滚 |

儿童和家长共享同一个小程序，但两类功能必须通过家长门槛区分。儿童不应直接进入账号、字库、提醒和数据删除设置。

### 2.2 年龄分层

年龄分层只用于推荐内容和默认流程，不构建复杂的算法自适应。

| 年龄档 | 默认字库 | 每日新字 | 默认步骤 | 交互原则 |
|---|---|---:|---|---|
| 3-4 岁 | 学前启蒙基础 | 2-3 个 | 字源动画、听音辨认、看图选择；描红为可选 | 以图片和声音为主，不强制阅读拼音和长文字 |
| 5-6 岁 | 学前启蒙进阶 | 3-5 个 | 字源动画、听音辨认、描红、图文测验 | 逐步加入拼音、组词和短句 |
| 6 岁以上/一年级 | 一年级教材同步 | 3-5 个 | 字源动画、认读、描红、测验、语境巩固 | 与教材单元对应，强调识字和基础书写 |

首次进入时询问“宝宝现在几岁”，并推荐字库。家长可以切换字库，但切换前显示：

- 新字库的适龄范围；
- 每日建议字数；
- 是否会保留当前学习进度；
- 切换后是否需要重新开始单元。

### 2.3 核心使用场景

**场景 A：首次使用**

登录/体验模式 → 家长确认年龄 → 推荐字库 → 家长确认每日字数 → 儿童开始第一个字。

**场景 B：日常学习**

进入首页 → 查看今日任务 → 优先完成到期复习 → 学习今日新字 → 查看结果 → 结束学习。

**场景 C：家长查看**

家长验证 → 查看今日新学、延迟回忆、待巩固汉字和学习时长 → 必要时进入单元复习或修改计划。

**场景 D：中断恢复**

学习中退出 → 保存当前字和步骤 → 下次进入显示“继续上次学习” → 从最近未完成步骤恢复。

## 三、产品目标与指标

### 3.1 MVP 产品目标

首版只验证以下假设：

1. 儿童愿意在低刺激、短时长的流程中完成一个字的学习；
2. 字源动画能够帮助儿童理解和记忆字形；
3. 次日复习能发现“完成学习但没有真正记住”的汉字；
4. 家长能通过简单报告理解孩子的学习状态；
5. 产品不会通过无限任务、复杂游戏或长时使用制造虚假留存。

### 3.2 指标定义

| 指标 | 定义 | MVP 观察方式 |
|---|---|---|
| 首字启动率 | 首次进入后点击开始第一个字的用户 / 首次进入用户 | 事件埋点 |
| 首字完成率 | 启动第一个字后完成全部必需步骤的用户 / 启动用户 | 事件埋点 |
| 单元完成率 | 开始单元后完成单元测试的用户 / 开始单元用户 | 服务端或本地记录 |
| 步骤流失率 | 某步骤开始后未进入下一步骤的比例 | 按步骤统计 |
| 即时正确率 | 学习结束时测验答对题数 / 总题数 | 按字、按单元记录 |
| 延迟回忆率 | 到期复习时首次答对的字数 / 到期复习字数 | 复习结果记录 |
| 复习完成率 | 实际完成的到期复习字数 / 到期复习字数 | 首页任务和复习记录 |
| D1/D7 留存 | 首次学习后次日/第 7 日再次完成有效学习的用户比例 | 登录和学习事件 |
| 家长报告查看率 | 进入家长报告并停留超过 5 秒的用户比例 | 家长模式事件 |

**推荐北极星指标**：每周完成延迟复习且仍能正确识别的汉字数。

“已学字数、星星数、连续打卡天数”保留为展示指标，但不能作为唯一的产品成功标准。

## 四、信息架构

```text
首页（今日任务）
├── 今日复习
├── 今日新字
├── 继续上次学习
└── 家长模式入口（需验证）

学习流程
├── 字源动画
├── 听音辨认
├── 描红/笔顺（按年龄档可选）
├── 图文或语境测验
├── 单元测试
└── 学习结果

复习
├── 今日到期复习
├── 单元复习
└── 常错字强化

图鉴
├── 已接触汉字
├── 已掌握汉字
├── 待巩固汉字
└── 汉字详情

家长模式
├── 宝宝档案与年龄
├── 字库与每日字数
├── 今日/本周学习报告
├── 学习时长与提醒
├── 隐私、数据导出/删除
└── 家长验证设置
```

底部导航仍保留 3 个 Tab：`今日`、`图鉴`、`我的`。`我的`中的设置和报告必须经过家长门槛；儿童可以看到“今日任务”和“图鉴”，但不应直接操作账号和内容配置。

## 五、首次使用流程

### 5.1 MVP 流程

1. 微信授权登录，或进入本地体验模式；
2. 展示未成年人使用提示和隐私摘要；
3. 家长确认宝宝年龄档；
4. 推荐适龄字库；
5. 家长选择每日新字数：2/3/5 个，年龄档提供默认值；
6. 进入首页；
7. 直接开始第一个字，首字流程不超过 3 分钟。

### 5.2 可跳过规则

- 家长可以暂不填写宝宝昵称，但年龄档必须确认；
- 可以跳过微信登录进入本地体验模式；
- 体验模式不承诺云端同步，页面必须明确提示；
- 用户可以稍后在家长模式修改年龄、字库和每日字数。

### 5.3 家长门槛

MVP 采用低成本家长验证：

- 长按按钮 2 秒；或
- 完成简单算术题；或
- 从屏幕指定方向滑动。

验证只保护设置和账号能力，不应阻止儿童完成当前学习步骤。

## 六、核心学习流程

### 6.1 单字流程

```text
开始单字
  ↓
字源动画
  ↓
听音辨认
  ↓
描红/笔顺（按年龄和设备能力）
  ↓
即时测验
  ↓
记录结果
  ↓
下一个字或单元测试
```

### 6.2 步骤一：字源动画

**目标**：建立字形与具体事物、动作或含义之间的联系。

**规则**：

- 默认时长 5-10 秒，禁止无意义循环；
- 动画结束后不自动跳转，显示“我看懂了，继续”按钮；
- 提供“再看一遍”和“播放讲解”按钮；
- 3-4 岁优先图片和语音，文字说明最多一行；
- 抽象字不能强行伪造字源，使用儿童可理解的意义说明；
- 每个动画必须有内容版本和审核状态。

**验收**：儿童不操作 10 秒后能看到完整演示；音频失败时仍可完成；重复播放不会重复计为掌握。

### 6.3 步骤二：听音辨认

MVP 采用点击选择，不实现 ASR 评分。

- 自动或手动播放目标读音；
- 提供 3 个大触控选项；
- 3-4 岁使用图片或目标字与干扰字组合；
- 5 岁以上可使用汉字或拼音选项；
- 答错后重新播放目标音，给出一次重新选择机会；
- 不将录音模拟描述为语音评测；
- 音频不可用时提供文字和图像兜底。

### 6.4 步骤三：描红/笔顺

- 5 岁以上默认开启；3-4 岁作为可选步骤；
- HanziWriter 或 Canvas 任一方案通过即可，不以具体库作为产品要求；
- 允许轻微偏差，避免儿童因笔画不准而反复受挫；
- 必须展示笔顺演示、起笔提示和清除重试；
- 描红完成只记录“完成书写练习”，不直接判定汉字已掌握；
- 设备不支持稳定触控时降级为笔顺动画，不阻断学习流程。

### 6.5 步骤四：即时测验

MVP 必做两类：

- 看字选图；
- 听音选字。

5 岁以上可增加人工审核的看图选字和语境选字，但必须作为可配置题型，不影响低龄儿童主流程。

规则：

- 每个字至少 1 道即时题；
- 题目选项随机排序，但目标字不能因位置固定被猜中；
- 答错记录题型、错误选项和当前步骤；
- 答错后显示解释或重播，不立即用高强度动画惩罚；
- 即时答对只能使状态达到“初步掌握”，不能直接标记“稳定掌握”。

### 6.6 单元测试

- 单元测试覆盖本单元全部汉字；
- 默认通过线为 80%；正确率达到 100% 获得 3 星，80%-99% 获得 2 星，低于 80% 获得 1 星且提示再次巩固；
- 单元通过只代表完成当前单元任务，不改变各汉字原有的掌握状态；
- 测试中答错的汉字统一进入“待巩固”，不能因单元总分通过而显示“全部掌握”；
- 测试结束显示：答对字、待巩固字、正确率、建议复习时间；
- 允许“稍后复习”，不强迫儿童立即重做完整单元；
- 通过单元测试不等于所有汉字都进入稳定掌握。

### 6.7 学习结果页

结果页展示：

- 本次接触字数；
- 即时答对字数和正确率；
- 待巩固字数；
- 下次复习时间；
- 今日学习时长；
- “结束学习”和“复习待巩固字”两个主操作。

不展示“再学 100 个字”“继续闯关”等无限延长学习时长的诱导文案。

## 七、学习状态与掌握规则

### 7.1 汉字状态

| 状态 | 进入条件 | 首页/图鉴展示 | 是否计入已掌握 |
|---|---|---|---|
| 未开始 | 未进入学习 | 灰色 + 问号 | 否 |
| 已接触 | 看过动画或播放过读音 | 浅色 | 否 |
| 初步掌握 | 完成即时测验且当次答对 | 彩色 + 初步标记 | 否 |
| 待巩固 | 即时测验或复习答错 | 橙色/提示标记 | 否 |
| 已掌握 | 即时答对，且至少一次延迟复习答对 | 彩色 + 掌握标记 | 是 |
| 稳定掌握 | 已掌握后再完成 2 次计划复习，且复习等级达到 4 | 深色/稳定标记 | 是 |

### 7.2 最小掌握判定

一个汉字只有同时满足以下条件，才进入“已掌握”：

1. 完成一次学习流程；
2. 即时测验答对；
3. 设备本地日期已经进入下一个自然日；
4. 延迟复习首次答对。

如果孩子在即时测验中答错，但在当天重试答对，状态最多为“初步掌握”，仍需经过延迟复习。

“稳定掌握”必须在进入“已掌握”后，再完成 2 次按计划到期的复习且均答对，同时复习等级达到 4。当天主动重复练习不计入这 2 次计划复习。

### 7.3 延迟回忆

次日或下一次打开应用时，优先抽取前一日新学字的 2-3 个进行回忆。回忆题不展示拼音，低龄档可以展示图片。

- 延迟答对：提升复习等级；
- 延迟答错：进入待巩固，次日再次出现；
- 连续两次答错：增加图片、字源或读音提示，不直接增加题量；
- 没有完成延迟复习时，不把字标记为稳定掌握。

## 八、复习与每日任务

### 8.1 复习等级

统一使用以下规则，前端、后端和文档不得各自定义一套算法：

| 等级 | 典型间隔 | 说明 |
|---:|---:|---|
| 0 | 当天/次日 | 首次学习或答错后 |
| 1 | 1 天 | 初次复习答对 |
| 2 | 3 天 | 连续复习答对 |
| 3 | 7 天 | 已形成短期记忆 |
| 4 | 15 天 | 中期巩固 |
| 5 | 30 天 | 稳定掌握前的抽查 |
| 6 | 60 天 | 长期抽查，可选 |

答对升一级，答错至少降一级，最低为 0。答错的字进入错题队列，并记录错误题型。

### 8.2 每日任务顺序

1. 先处理到期复习；
2. 今日复习默认上限 10 个字，超出部分顺延；
3. 复习完成后开放今日新字；
4. 新字默认 3 个，家长可设置为 2/3/5 个；
5. 未完成的新字不惩罚连续天数，不要求一次性补齐；
6. 学习达到建议时长后展示“今天完成”；
7. 家长可以手动进入单元复习，但不会改变儿童当日任务上限。

### 8.3 复习模式

默认采用轻量闪卡：

- 正面显示大字或图片，不显示拼音；
- 点击翻转显示拼音、组词和读音；
- 底部显示“认识”和“不认识”；
- 左右滑动仅作为辅助，不作为唯一操作方式；
- 所有卡片都提供音频按钮；
- 复习结束显示答对、答错和下次复习时间。

单元复习和常错字强化可以复用该模式，但必须区分复习范围。

## 九、家长模式与儿童保护

### 9.1 家长模式范围

以下入口必须经过家长验证：

- 修改年龄、昵称和头像；
- 切换字库；
- 修改每日新字数；
- 设置提醒和学习时长；
- 查看完整学习报告；
- 账号退出、数据删除和隐私设置。

### 9.2 家长报告

今日报告至少展示：

- 今日新学字；
- 今日完成复习字；
- 即时答对率；
- 延迟回忆率；
- 待巩固字列表；
- 学习时长和完成次数。

报告文案使用事实描述，例如“今天有 2 个字需要巩固”，避免使用“表现超过多少孩子”等无依据的竞争性表述。

### 9.3 时长和提醒

- 默认单次学习建议 10-15 分钟；
- 到达建议时长后提供结束选项，不强制退出；
- 提醒时间默认关闭，开启前说明需要微信通知授权；
- 家长可以关闭提醒；
- 连续打卡只作为弱激励，不因中断而对儿童使用负面惩罚。

### 9.4 隐私与数据

- 登录前展示隐私摘要，说明昵称、头像、学习记录和宝宝信息的用途；
- 体验模式明确说明数据仅保存在本地；
- 支持查看、修改和删除账号及学习数据；
- 不采集与学习无关的儿童敏感信息；
- 不在儿童界面展示付费、外链或社交诱导；
- 数据同步冲突时以可解释规则合并，并在家长模式显示最近同步时间。

## 十、汉字内容规范

### 10.1 内容字段

每个汉字至少包含：

```javascript
{
  _id: "火",
  pinyin: "huǒ",
  char_type: "象形",
  age_range: [3, 6],
  difficulty: 1,
  radical: "火",
  strokes: 4,
  stroke_order: "4334",
  teaching: {
    animation_type: "origin",
    explanation: "像燃烧的火焰",
    audio_asset: "audio/huo.mp3",
    image_asset: "images/huo.png",
    source_note: "教学化解释，不等同于严格字源考据"
  },
  examples: {
    words: ["火车", "大火"],
    sentence: "红红的火真暖和。"
  },
  confusable_chars: ["大", "太"],
  review_status: "approved",
  content_version: 1,
  reviewer: "content-editor"
}
```

### 10.2 内容审核清单

每个字上线前确认：

- 读音、拼音和声调正确；
- 字源说明没有把教学类比冒充为严格历史结论；
- 动画最终字形正确；
- 例词符合年龄和生活经验；
- 例句长度适合朗读；
- 图片无版权和敏感内容风险；
- 干扰项难度合理，不靠位置或颜色猜答案；
- 音频在静音、失败和重复播放时都有可用反馈；
- 内容版本可以回滚。

## 十二、MVP 优先级与上线门槛

### 12.1 P0：上线阻塞

- 微信登录和本地体验模式；
- 年龄确认和适龄字库推荐；
- 今日任务：先复习、后新学；
- 字源动画；
- 听音三选一；
- 描红或笔顺动画降级方案；
- 即时测验；
- 延迟回忆；
- 学习状态和复习等级统一；
- 家长门槛；
- 基础家长报告；
- 本地和云端进度同步；
- 隐私摘要、数据删除入口；
- 至少 2 个可完整体验的单元；
- 真机验证音频、触控、返回和断网恢复。

### 12.2 P1：首版后迭代

- 完整闪卡复习体验；
- 更多年龄档内容；
- 阅读语境练习和人工审核短句；
- 每日学习量和提醒设置完善；
- 字库和汉字内容后台化；
- 学习报告趋势和家长周报；
- 打卡海报和分享；
- 内容版本管理工具。

### 12.3 P2：有数据后再做

- ASR 跟读评分；
- AI 造句，但必须有儿童内容安全和人工审核策略；
- 星星商店和复杂兑换；
- 多儿童档案；
- 自适应步骤和个性化推荐；
- 班级、战队和社交功能；
- 等级、称号和头像框体系。

### 12.4 上线 Gate

产品功能完成不代表可以上线，必须同时满足：

1. 首次用户可以在 3 分钟内完成第一个字；
2. 年龄、字库和每日任务状态前后一致；
3. 即时掌握和延迟掌握可以区分；
4. 复习规则在前端、后端和报告中一致；
5. 家长可以查看待巩固字和学习时长；
6. 儿童无法直接修改家长设置和删除数据；
7. 音频失败、断网、退出和恢复均有明确反馈；
8. 内容审核清单覆盖所有上线汉字；
9. 核心事件可以统计首字完成、延迟回忆和复习完成；
10. `type-check`、构建、测试、e2e 和真机回归通过；
11. 生产密钥、隐私文案和服务器配置完成脱敏与复核；
12. 字库状态、原型、代码和验收报告使用同一份基线。

## 十三、产品验收清单

### 13.1 儿童体验

- [ ] 首屏可以理解今天要做什么；
- [ ] 关键按钮触控区不小于 88rpx；
- [ ] 低龄档不依赖长文字；
- [ ] 每次操作有即时视觉或声音反馈；
- [ ] 答错不会被羞辱或强制重复过多次；
- [ ] 达到建议时长后可以自然结束；
- [ ] 返回和中断不会丢失进度；
- [ ] 音频失败时有可操作的兜底。

### 13.2 学习有效性

- [ ] 每个字都有即时测验结果；
- [ ] 即时答对不会直接等同于稳定掌握；
- [ ] 次日或下次有效复习会验证延迟回忆；
- [ ] 答错字进入待巩固和错题队列；
- [ ] 复习等级和下次复习时间可解释；
- [ ] 图鉴能区分已接触、已掌握和待巩固。

### 13.3 家长和合规

- [ ] 年龄和字库选择经过家长确认；
- [ ] 设置、报告和数据删除有家长门槛；
- [ ] 隐私摘要在首次使用前可见；
- [ ] 体验模式与云端模式边界明确；
- [ ] 可以删除账号和学习数据；
- [ ] 儿童界面无未经授权的付费或外链诱导；
- [ ] 提醒和学习时长可关闭或调整。

### 13.4 内容质量

- [ ] 读音、拼音、笔顺和字形经过审核；
- [ ] 字源动画说明有来源或教学化声明；
- [ ] 图片、音频和例句经过年龄适配；
- [ ] 干扰项不会依赖位置、颜色或题型漏洞；
- [ ] 内容版本可以追踪和回滚。

## 十五、版本结论

v3.1 继续坚持“轻量、克制、字源差异化”的方向，但把产品成功标准从“孩子完成了多少流程”调整为“孩子在合理时长内学到并记住了多少字，家长能否清楚理解结果”。

首版最重要的不是增加游戏、社交或 AI 功能，而是建立可信的学习闭环：

```text
适龄内容
  → 短时学习
  → 即时检验
  → 延迟回忆
  → 待巩固复习
  → 家长可读报告
  → 继续适量学习
```

只要这条链路能够稳定运行，字源动画就有机会成为真正的产品差异化；如果只能证明孩子点完了按钮，产品仍然只是一个完成度较高的识字交互 Demo。
