# 个人开发者启动指南：趣字宝 (Little Hanzi)

作为个人开发者，在正式编写代码前，您需要完成以下四个维度的准备工作。这份清单旨在帮助您规避常见的“坑”，确保项目顺利启动。

## 1. 技术选型与环境搭建 (Tech Stack)

对于个人开发者，**效率**和**成本**是首要考虑因素。

### 1.1 前端框架 (小程序)
*   **推荐方案**：**Uni-app** (Vue.js 语法)
    *   *理由*：一套代码可同时发布到微信小程序、iOS、Android。生态成熟，插件丰富。
    *   *备选*：原生微信小程序开发 (如果只做微信端，原生性能最好)。
*   **开发工具**：
    *   HBuilderX (Uni-app 官方IDE) 或 VS Code (需装插件)。
    *   微信开发者工具 (必须安装，用于调试和上传)。

### 1.2 后端服务 (Backend)
*   **推荐方案**：**微信云开发 (WeChat CloudBase)**
    *   *理由*：**强烈推荐**。免运维（Serverless），免域名备案，直接在前端调用数据库，每月有免费额度，极度适合个人开发者。
    *   *备选*：Node.js (Koa/Express) + MongoDB/MySQL (需购买服务器 + 域名备案，周期长，成本高)。

### 1.3 AI 能力接入 (Core Features)
我们需要两个核心 AI 能力：
1.  **大语言模型 (LLM)** - 用于“魔法造句”：
    *   **推荐**：**通义千问 (Qwen)** 或 **智谱AI (GLM)**。
    *   *理由*：中文理解能力强，API 价格亲民（甚至有免费额度），响应速度快。
2.  **语音合成 (TTS)** - 用于“AI 领读”：
    *   **推荐**：**Azure TTS** (微软) 或 **阿里云 TTS**。
    *   *理由*：Azure 的神经网络语音（Neural TTS）目前最接近真人，有专门的“童声”音色，非常适合儿童产品。

## 2. 账号与资质准备 (Accounts & Licenses)

这是最容易卡住个人开发者的一环，请提前办理。

### 2.1 微信小程序账号
*   **注册**：前往 [微信公众平台](https://mp.weixin.qq.com/) 注册小程序账号。
*   **类目选择**：
    *   *避坑*：个人主体**不能**做“教育”类目（会被要求提供办学资质）。
    *   *建议*：选择 **“工具 > 效率”** 或 **“工具 > 信息查询”**。在提审时，尽量弱化“课程、教学”字眼，强调“汉字查询、阅读辅助工具”属性。
*   **名称抢注**：先搜索“趣字宝”是否被占用，尽早确定名称。

### 2.2 软著 (软件著作权)
*   *现状*：目前微信小程序上架部分类目可能需要软著。
*   *准备*：如果打算长期运营或接广告，建议在阿里云/淘宝找代理申请软著（费用约 300-600 元，周期 1-2 个月）。MVP 阶段可先尝试不上架，用“体验版”给种子用户测试。

### 2.3 第三方 API 账号
*   **阿里云/Azure**：注册并实名认证，申请 TTS 和 LLM 的 API Key。
*   **七牛云/OSS**：如果云开发存储不够用，可能需要对象存储来放图片/音频资源。

## 3. 素材与资源准备 (Assets)

儿童产品对“颜值”和“声音”要求极高。

### 3.1 美术资源
*   **UI 组件**：寻找一套适合的开源图标库（如 iconfont.cn）。
*   **字源素材**：这是核心壁垒。需要准备“汉字 -> 象形图 -> 实物图”的演变素材。
    *   *低成本方案*：购买商用授权的矢量图库，或者使用 AI 绘画工具（Midjourney/Stable Diffusion）辅助生成风格统一的插画。
*   **字体**：必须使用**免费商用字体**（如：庞门正道标题体、站酷快乐体、思源宋体），严禁使用方正等版权字体，否则会收律师函。

### 3.2 音频资源
*   **音效**：点击声、正确/错误提示音、背景轻音乐。可在“爱给网”等素材站寻找免费商用素材。

## 4. 开发路线图 (Roadmap)

建议分三步走，不要试图一口气做完。

### 第一阶段：MVP (最小可行性产品) - 周期：2周
*   **目标**：跑通核心流程，验证孩子喜不喜欢。
*   **功能**：
    *   首页地图（做 3 个关卡即可）。
    *   核心学习流（字源图展示 + 系统发音 + 简单描红）。
    *   无后端，数据写死在本地 JSON 中。

### 第二阶段：云端与交互 - 周期：2周
*   **目标**：引入 AI，增加粘性。
*   **功能**：
    *   接入微信云开发，记录用户进度。
    *   接入 AI API，实现“魔法造句”。
    *   增加“跟读打分”（需接入微信同声传译插件）。

### 第三阶段：运营与裂变 - 周期：长期
*   **目标**：拉新。
*   **功能**：
    *   打卡海报生成。
    *   积分商城。
    *   支付功能（如果计划卖课，需升级为企业主体）。

---

## 5. 避坑小贴士
1.  **不要过早优化**：初期代码写得乱一点没关系，先把功能跑通。
2.  **不要为了 AI 而 AI**：AI 只是辅助，核心还是识字体验。如果 AI 生成的句子不通顺，宁可人工预设几个好句子。
3.  **注意合规**：收集用户语音和录音时，必须在隐私协议中明确告知，否则小程序审核过不了。
