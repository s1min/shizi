# 学习详情页 UI 优化设计

## 背景
当前学习详情页首屏包含“字源演变”说明，并且顶部进度条使用的是步骤进度。对于学龄前儿童，这类抽象解释理解成本高，顶部信息也不够直观。本次优化目标是让页面更贴近低龄识字场景：先看当前学哪个字、现在做到哪一步，再通过更有亲和力的组词卡片强化记忆。

## 目标
- 去掉首屏“字源演变”信息块，减少抽象认知负担。
- 将顶部进度改为“当前学习到第几个生字 / 总生字数”。
- 在顶部中间明确显示当前步骤名称。
- 将左上角退出入口改为更儿童化的返回按钮，仅显示 `wd-icon` 的 `arrow-left`。
- 将“常用组词”改为糖豆小卡片风格，提升亲和力与趣味性。

## 非目标
- 不调整学习流程顺序，仍保留 origin / speak / trace / quiz / complete 这几个阶段。
- 不改动学习数据来源与进度持久化逻辑。
- 不引入复杂互动动画或新的学习模块。

## 设计方案

### 1. 页面结构
学习详情页维持现有分步骤学习结构，但重新组织信息层级：
1. 顶部导航区：返回按钮、步骤名称、生字进度。
2. 主内容区：汉字主体、拼音、发音按钮。
3. 辅助认知区：常用组词等直观内容。
4. 底部主按钮：继续下一步。

`origin` 步骤仍作为首个认知步骤存在，但不再展示“字源演变”说明卡，避免向学龄前儿童暴露难理解的象形/演变解释。

### 2. 顶部导航区
顶部从“步骤 chip + 退出学习按钮 + 步骤进度条”调整为“返回按钮 + 居中步骤名 + 生字进度条”：
- 左侧为圆润、轻量、活泼的返回按钮，内部仅放 `wd-icon name="arrow-left"`。
- 中间为纯文字步骤名，例如“字形认知”“跟读练习”“描红练习”“趣味小测”。不再保留步骤 chip。
- 下方进度条按当前生字序号计算，而不是按步骤总数计算。
- 进度条右侧数字显示 `当前字序/总字数`，例如 `2/7`。
- 进度文案与数字保持一致，突出“已经学到第几个字”。

### 3. 首屏认知内容
首屏继续保留：
- emoji/图示圆球
- 当前汉字主体
- 拼音与发音按钮
- 常用组词
- 底部继续按钮

首屏移除：
- “字源演变 / 字形拆解 / 字义理解”说明卡
- 对应的标题、标签、分隔线与解释文案

这样首屏聚焦在“看字、听音、认词、继续”，减少分散注意力的信息。

### 4. 常用组词样式
“常用组词”改为糖豆小卡片风格：
- 更大的圆角与更饱满的内边距。
- 浅暖色底搭配细描边，保持暖白橙黄体系。
- 柔和阴影，视觉上像儿童贴纸或糖豆标签。
- 保持规则整齐排列，不引入手账式错落布局。
- 无需复杂动画，只保留轻量触感和静态趣味。

空状态样式保持友好，但与组词卡片视觉语言统一。

## 交互与状态
- 返回按钮继续复用现有退出确认逻辑：点击后弹出“是否退出”确认框，并保存当前进度。
- 生字进度基于 `charIndex + 1` 与 `unitChars.length` 计算。
- complete 状态下顶部仍展示当前步骤名与当前字序，保证信息连续性。

## 需要修改的代码点
- `shizi-frontend/src/subpkg-learning/learn/index.vue`
  - 调整顶部结构与进度计算逻辑。
  - 将步骤文案展示从左侧 chip 移到顶部中间。
  - 将进度条从步骤进度改为生字进度。
  - 将退出按钮改为 `wd-icon arrow-left` 风格入口。
- `shizi-frontend/src/subpkg-learning/components/learn/CharCard.vue`
  - 删除说明卡区域及相关计算逻辑。
  - 重新调整组词区域样式，改为糖豆小卡片风格。
  - 保持主内容、按钮和现有动画节奏基本稳定。

## 测试与验收
- 页面进入后，顶部右侧数字应显示当前生字进度，而非步骤进度。
- 顶部中间应正确显示当前步骤名称。
- 左上角退出入口应为 arrow-left 图标按钮，并仍可正常触发退出确认。
- 首屏不再出现“字源演变”或类似解释块。
- 常用组词在视觉上明显比当前更圆润、更饱满、更有儿童感。
- 切换生字和步骤时，不影响原有学习流程与跳转。
