# 学习详情页顶部真机风险预防型微调设计

## 背景
学习详情页顶部导航已经完成两轮优化：先完成信息结构调整，再完成更软更萌的视觉微调。当前这轮不再追求更强装饰感，而是针对真机和小程序环境下更容易暴露的问题做预防型微调，让现有方案在不同机型上更稳、更顺手。

## 目标
- 保持当前顶部布局、交互和视觉方向不变。
- 提前规避真机环境下的热区、视觉居中、间距和阴影问题。
- 在不增加复杂度的前提下，让顶部更稳、更自然。

## 非目标
- 不改动学习流程和进度逻辑。
- 不改动模板结构与现有信息层级。
- 不继续增强糖感或新增装饰元素。
- 不修改首屏主体、组词区和底部按钮。

## 设计方案

### 1. 改动边界
本轮是“风险预防型微调”，不是新一轮视觉重做：
- 只动顶部导航区。
- 只改样式，不改结构和逻辑。
- 所有微调都服务于“真机更稳”，而不是“视觉更夸张”。

### 2. 返回按钮热区
返回按钮继续保持现有视觉尺寸和泡泡糖按钮方向，但重点从“看起来更萌”转为“点起来更稳”：
- 保持当前按钮视觉大小，不继续放大。
- 优先通过按钮所在栅格宽度、对齐和外部留白，确保点击区域稳定。
- 避免真机上出现按钮看起来不小，但手指落点仍偏挤的问题。

### 3. 标题视觉居中
当前顶部使用三列布局：左按钮、中标题、右数字。这个结构在真机上可能出现“数学居中但视觉偏心”的问题，尤其是在右侧数字长度变化时：
- 微调左右两侧的占位平衡，让中间标题更稳定地维持视觉居中。
- 优先保证在 `2/7`、`10/28` 这类不同长度数字下标题都不容易偏移。
- 不改变标题为纯文字的设定。

### 4. 进度条与间距
进度条已经具备果冻条方向，这一轮主要做克制性修正：
- 保留圆润、柔和的视觉方向。
- 略微收敛阴影和高光，避免部分安卓机型上显灰、显脏。
- 微调标题、进度条和容器底部之间的垂直节奏，让顶部呼吸感更自然。
- 保证顶部在高状态栏机型下也不显挤。

### 5. 层次控制
这一轮不再继续叠加视觉效果，而是做“减法式精修”：
- 保留层次，但收掉多余的浮感。
- 按钮和进度条都维持亲和力，但避免在真机上变得过厚、过亮、过糊。
- 整体目标是“稳、顺手、自然”。

## 需要修改的代码点
- `shizi-frontend/src/subpkg-learning/learn/index.vue`
  - 仅调整顶部导航相关 SCSS。
  - 重点微调 grid 列宽、按钮可点稳定性、阴影浓度、顶部间距与进度条细节。
  - 不改模板，不改脚本逻辑。

## 测试与验收
- 顶部结构保持不变：左按钮、中标题、右数字、下方进度条。
- 返回按钮视觉体积不需要继续放大，但触控感应更稳。
- 标题在不同进度数字长度下仍尽量保持视觉居中。
- 进度条仍保留果冻条感，但阴影和高光不过重。
- 顶部整体在真机/小程序里更顺眼，不显挤、不显脏。
