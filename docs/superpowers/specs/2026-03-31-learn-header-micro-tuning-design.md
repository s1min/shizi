# 学习详情页顶部导航微调设计

## 背景
学习详情页已完成基础 UI 优化：顶部改为返回按钮、步骤名称和生字进度，首屏说明卡已移除，组词区也更适合低龄儿童。当前这一轮不再调整结构与信息层级，而是在既有方案上继续细化顶部视觉，让导航区更软、更萌、更贴近儿童识字产品的气质。

## 目标
- 保持当前顶部信息结构不变。
- 将返回按钮从“功能按钮感”进一步微调为“泡泡糖按钮感”。
- 让顶部标题更稳定地成为视觉中心。
- 让进度条更像圆润、柔和的果冻条。
- 保持右侧数字清爽，不增加额外装饰。

## 非目标
- 不改动学习流程、步骤命名与进度计算逻辑。
- 不新增新的顶部元素或装饰图标。
- 不改动首屏主体、组词区和底部按钮。

## 设计方案

### 1. 顶部整体气质
顶部继续使用当前卡片式导航容器，但整体气质从“功能型 header”再往“儿童引导区”推进一步：
- 保持原有布局：左侧返回按钮，中间步骤名，右侧进度数字，下方进度条。
- 通过圆角、留白、边框和阴影的细化，让顶部更柔和。
- 不增加复杂装饰，避免影响识字主任务。

### 2. 返回按钮
返回按钮继续只展示 `wd-icon arrow-left`，但视觉上进一步软化：
- 按钮尺寸略增，触感更饱满。
- 边框颜色更浅，弱化功能控件感。
- 底色更奶、更暖，形成儿童产品常见的柔软按钮观感。
- 阴影更蓬松、更浅，不使用明显硬边界。
- 按下态依然保留轻微缩放与颜色变化，增强触感。

### 3. 标题
中间步骤名称仍保持纯文字，不增加背景底牌：
- 通过字重、颜色和垂直对齐微调，让标题更稳定地占据视觉中心。
- 颜色保持暖棕调，但比说明文字更聚焦。
- 与左侧返回按钮形成“一个活泼、一个稳定”的平衡关系。

### 4. 进度数字
右侧数字继续保持朴素数字表达，例如 `2/7`：
- 不做徽章包裹，不增加额外底色。
- 通过轻微调柔颜色和字重，使其更亲和，但仍保持可读性。
- 保持其作为辅助信息的层级，不抢标题视觉中心。

### 5. 进度条
进度条在本轮是主要微调重点：
- 条高略增，使其更像儿童产品里的果冻条而不是功能细线。
- 圆角进一步拉满，增强软糯感。
- 轨道底色更奶白、更柔和。
- 填充色延续暖橙黄，但过渡更顺、更奶，减少硬朗高对比感。
- 整体节奏与标题、数字之间更协调，让顶部看起来是一张完整的小卡片。

## 需要修改的代码点
- `shizi-frontend/src/subpkg-learning/learn/index.vue`
  - 仅调整顶部导航相关 SCSS。
  - 必要时微调顶部容器的 padding / gap / 尺寸。
  - 不改模板结构，不改脚本逻辑。

## 测试与验收
- 顶部布局结构保持不变，功能行为保持不变。
- 返回按钮视觉明显更软、更萌，但仍然清晰可点。
- 中间标题依旧是纯文字，并成为顶部视觉中心。
- 右侧数字仍是纯数字显示，无徽章底。
- 进度条明显比现在更厚、更圆、更柔和。
- 不影响顶部在真机/小程序中的对齐与点击区域。
