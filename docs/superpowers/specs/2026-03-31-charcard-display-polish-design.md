# CharCard 字体与拼音区精修设计

## 背景
当前学习详情页已经完成顶部、词组卡片和底部按钮的多轮优化。新的反馈集中在 `CharCard.vue` 首屏展示层级：汉字本体还可以更大一些；拼音区中间的播放按钮还不够活泼，风格希望更贴近页面里已经建立起来的糖豆按钮语言；同时，组词项现在略显偏大，可以整体缩一号，让首屏主角重新回到汉字本身。

## 目标
- 放大 `char-display` 中的主汉字，让首屏视觉焦点更集中。
- 将 `char-pinyin` 中的 `speaker-button` 调整为更活泼的糖豆按钮风格。
- 将组词项整体缩小一号，减弱对主汉字的抢占。

## 非目标
- 不修改学习流程、步骤跳转或按钮行为。
- 不修改词组点击播音逻辑。
- 不重做 `CharCard.vue` 的结构层级，只做局部视觉精修。
- 不新增新的动画阶段或复杂动效系统。

## 设计方案

### 1. 汉字本体放大
保留当前 `char-display` 结构和居中方式，只调整视觉比例：
- `char-main` 字号再放大一档，让首屏第一眼先看到字。
- 如果放大后上下呼吸感变紧，只做很小幅度的间距微调，不引入新的布局结构。
- 保持现有字体、字重和整体气质，避免因为过度装饰削弱识字清晰度。

### 2. 拼音区糖豆按钮化
`char-pinyin` 继续保持“按钮 + 拼音文本”的横向组合，但按钮风格升级：
- `speaker-button` 使用更圆润、偏暖色、略带渐变和轻阴影的糖豆按钮方向。
- 按钮需要保留明确的按下反馈，让孩子知道这是可点击区域。
- 按钮体量可适度增大，但不能压过汉字本体和拼音文本。
- `speakChar()` 逻辑保持不变，只改视觉呈现。

### 3. 组词项缩一号
组词区继续沿用上一轮的多主题浅色方案和整词点击发音：
- `word-item` 的字号、内边距、圆角和阴影都同步缩一档。
- 缩小后仍要保留轻量点击反馈，确保既活泼又不难点。
- 目标是让词组区更像辅助信息，而不是与主字争主角。

### 4. 改动边界
本轮只修改一个文件：
- `shizi-frontend/src/subpkg-learning/components/learn/CharCard.vue`
  - 放大 `char-main`
  - 精修 `char-pinyin` / `speaker-button`
  - 缩小 `word-item`

## 交互与状态
- 保持现有 `playAnimation()` 节奏不变。
- 保持 `speakChar()`、`playWord()`、`handleNext()` 业务行为不变。
- 保持现有多主题词组卡片和整词发音。

## 测试与验收
- 主汉字比当前明显更大，首屏视觉中心更明确。
- `speaker-button` 更活泼、更接近页面现有糖豆按钮风格。
- 拼音区域仍清晰，不显拥挤。
- 组词项较当前缩小一号，但文字依旧清楚、点击区域依旧可用。
- 点击发音、词组播放和“我记住了，继续”逻辑均保持正常。
