# 学习详情页样式二次调整设计

## 背景
当前学习详情页已经完成基础 UI 优化，包括顶部进度重构、说明卡移除、组词糖豆卡片化，以及多轮顶部视觉微调。最新反馈聚焦在 4 个更具体的体验问题：顶部信息仍可更贴近微信胶囊按钮所在水平线；进度数字与进度条需要整合到同一行；常用词组希望通过更丰富但淡雅的多主题背景吸引儿童点击，并支持点击播放整个词组读音；底部“我记住了，继续”按钮存在视觉不居中的问题。

## 目标
- 让顶部卡片内部上边距更紧凑，使返回按钮与步骤名更贴近微信胶囊按钮所在行。
- 将当前进度数字与进度条整合到同一行显示。
- 将常用词组做成多主题浅色卡片，并支持点击播放整个词组读音。
- 修正底部主按钮的视觉居中问题。

## 非目标
- 不改动学习流程和步骤跳转。
- 不改动学习数据结构与进度计算逻辑。
- 不新增新的学习模块或复杂动画系统。
- 不重做整页布局，只在现有结构上精修。

## 设计方案

### 1. 顶部结构微调
顶部不改变外部位置，只压缩卡片内部上边距：
- 返回按钮和步骤名称更贴近微信胶囊按钮一行。
- 顶部从当前的“标题行 + 进度条行”进一步收敛为“标题行 + 进度行”两层结构。
- 当前进度数字从标题行移到进度条同行，增强阅读一致性。

### 2. 进度信息整合
进度条与数字整合到同一行：
- 左侧为圆润的进度条。
- 右侧为当前字序 / 总字数，例如 `3/7`。
- 进度条和数字形成同一组信息，视觉上比现在更直观。

### 3. 常用词组多主题卡片
词组区从统一色卡片升级为多主题浅色卡片：
- 使用 6~8 组儿童友好、很淡的浅色主题顺序轮换。
- 同一屏内尽量减少重复，增强活泼感。
- 保持整体色彩干净，不用高饱和颜色。
- 每个词组卡片可点击，点击后播放整个词组读音，而不是只播当前单字读音。
- 点击态保留轻量反馈，让孩子知道“这个能点”。

### 4. 底部主按钮居中修正
“我记住了，继续”按钮的处理重点是视觉居中，而不是只依赖 `width: 100%`：
- 检查按钮所在容器左右 padding 是否一致。
- 确保按钮自身宽度、阴影和圆角不会造成视觉偏心。
- 必要时通过按钮容器的布局方式（例如显式居中）来修正，而不是只放大宽度。

### 5. 实现边界
本轮只修改两个文件：
- `shizi-frontend/src/subpkg-learning/learn/index.vue`
  - 顶部内部上边距压缩
  - 标题行 / 进度行重组
  - 数字与进度条同排
- `shizi-frontend/src/subpkg-learning/components/learn/CharCard.vue`
  - 多主题词组卡片
  - 点击播放整个词组读音
  - 底部按钮视觉居中修正

## 交互与状态
- 点击词组卡片时，调用现有语音能力播放整个词组文本。
- 若词组为空，继续保持当前空状态文案。
- 不改变“我记住了，继续”按钮的业务行为，只修正显示与布局。

## 测试与验收
- 顶部返回按钮和步骤名应明显更贴近微信胶囊按钮所在高度。
- 进度条与 `3/7` 这类数字在同一行显示。
- 词组卡片同屏呈现多种淡彩主题，不显杂乱。
- 点击任一词组卡片，会播放整个词组读音。
- 底部“我记住了，继续”按钮视觉上居中，不再出现偏左/偏右的观感。
- 学习流程、步骤切换和下一步逻辑保持不变。
